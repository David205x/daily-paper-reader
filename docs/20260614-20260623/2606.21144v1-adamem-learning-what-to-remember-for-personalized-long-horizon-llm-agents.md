## 摘要
大型语言模型（LLM）智能体的长期记忆系统通常试图“记住一切”，统一提取记忆以尽可能多地保留事实。然而在生产环境中，推理成本和有限的上下文预算使得这一做法难以为继：除了将原始对话整合到记忆中，智能体必须施加“写入控制”，高效地仅保留每个用户真正关心的信息。否则，长时程个性化交互会遭受“记忆膨胀”，无关琐事挤占有用信息，并逐渐降低问答准确率。我们认为，值得记住的内容是角色依赖的，并提出了AdaMem（自适应记忆），一种从反馈中学习为每个用户“记住什么”的方法。AdaMem维护一个结构化的、特定角色的记忆策略，并通过每周问答反馈以轻量级、补丁式的自我反思步骤（含失败回滚）对其进行优化。为了研究这一设置，我们构建了AdaMem-Bench，一个模拟数周交互并逐周问答的基准测试。在两种提取模型和两种反馈模式下，AdaMem相比于统一的Mem0基线，问答准确率最高提升+9.0%，同时记忆容量缩小9%。

## Abstract
Long-term memory systems for Large Language Model (LLM) agents typically try to \emph{remember everything}, extracting memories uniformly to retain as many facts as possible. In production, however, inference cost and finite context budgets make this untenable: beyond consolidating raw dialogue into memory, an agent must exert \emph{write control}, efficiently keeping only the information each user actually cares about. Otherwise, long-horizon personalized interactions suffer \emph{memory bloat}, where irrelevant trivia crowds out useful information and steadily erodes question-answering (QA) accuracy. We argue that what is worth remembering is role-dependent, and propose \textbf{AdaMem} (Adaptive Memory), a method that \emph{learns what to remember} for each user from feedback. AdaMem maintains a structured, role-specific Memory Policy and refines it from weekly QA feedback through a lightweight, patch-style self-reflection step with failure rollback. To study this setting, we build \textbf{AdaMem-Bench}, a benchmark that simulates weeks of interaction with week-by-week QA. Across two extraction models and two feedback modes, AdaMem improves QA accuracy by up to \textbf{+9.0\%} over the uniform Mem0 baseline while shrinking memory volume by \textbf{9\%}.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有LLM智能体的长期记忆系统（如Mem0、MemoryBank）采用统一提取规则，试图“记住一切”，将所有事实性陈述存入记忆库。这导致在长时程个性化交互中产生**记忆膨胀（memory bloat）**：大量无关琐事（如日常问候、次要日程）在检索时挤占有用信息，逐步降低问答准确率。
- **根本矛盾**：真实部署中推理成本和有限上下文预算迫使智能体不能无限制记忆；关键在于“写入控制”——只保留每个用户真正关心的信息。信息的重要性是**角色依赖的**（role-dependent）：与管理者的对话应跟踪业务决策，与健身伙伴的对话应跟踪锻炼计划。因此需要一种机制来**学习什么值得记住**。
- **整体含义**：将长期记忆提取从被动、统一的过程转变为主动、偏好对齐的过程，通过反馈持续优化记忆策略，从而在工业级成本约束下提升个性化问答性能和内存效率。

## 2. 论文提出的方法论

### 核心思想
- **AdaMem（自适应记忆）**：维护一个结构化的、角色特定的**记忆策略（Memory Policy）**，并通过每周QA反馈进行**补丁式自我反思（patch-style self-reflection）**，以决定每个用户应记住哪些信息。策略更新附带**失败回滚（failure rollback）**机制，确保稳定性。

### 关键技术细节
1. **记忆策略结构**：策略 P = {C₁: T₁, ..., Cₙ: Tₙ}，是一个JSON对象，将角色字符映射到自然语言提取偏好（如“只提取最终商业决策和截止日期，忽略日常闲聊”）。
2. **闭环周循环流程**（Algorithm 1）：
   - **每日提取**：基于当前策略，将对话中的相对时间引用解析为绝对日期（时间锚定），提取偏好记忆存入数据库。
   - **每周评估**：使用检索增强回答每周QA问题，获取反馈。
   - **反馈诊断**：识别错误答案，判定失败类型为**召回失败**（策略太严格，遗漏信息）或**精度失败**（策略太宽松，噪声干扰）。
   - **策略更新**：生成策略补丁ΔP，若在历史QA上验证后性能下降则回滚，否则更新策略。
   - **QA写回**：用更新后记忆重新生成本周答案并写回。
3. **补丁式反思**：针对错误反馈生成自然语言规则补丁，而非重新训练模型。
4. **时间锚定**：在提取提示中注入会话绝对日期，要求LLM将“明天”等相对时间解析为绝对日期后再存储。
