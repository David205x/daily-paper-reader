## 摘要
语言模型推理可以通过在测试时使用扩展推理计算的支架来显著改进，这些计算跨越不同的原语——单次推理轨迹内的顺序推理、独立采样的并行轨迹以及将多个推理轨迹聚合成最终响应。然而，在后训练阶段，语言模型仅针对单次轨迹内的顺序推理进行优化。我们提出了顺序-并行-聚合强化学习（SPIRAL），这是一个框架，其中语言模型被训练使用所有三种原语，作为统一推理计算管线的一部分。具体来说，语言模型首先并行采样一组独立轨迹，每个轨迹通过顺序思维链推理产生，然后基于这些轨迹生成最终的聚合轨迹；所有组件都针对最终聚合响应的奖励进行端到端优化。为了训练这一系统，SPIRAL使用集合强化学习来教会模型产生一组对聚合器整体有用的轨迹，并使用标准强化学习来教会模型将集合聚合为改进的最终响应。我们在推理任务上的实验表明，SPIRAL能有效随推理计算扩展，当三种计算原语都扩展时，其性能比GRPO高出最多11倍的扩展效率和15%的性能提升。

## Abstract
Language model reasoning can be substantially improved at test time via scaffolds that scale inference compute across different primitives -- sequential reasoning within a trace, independently sampled parallel traces, and aggregation of multiple reasoning traces into a final response. During post-training, however, language models are optimized only for sequential reasoning within a single trace. We introduce Sequential-Parallel-Aggregative Reinforcement Learning (SPIRAL), a framework in which a language model is trained to use all three primitives, as part of a unified inference compute pipeline. Concretely, the language model first samples a set of independent traces in parallel, each produced through sequential chain-of-thought reasoning, and then generates a final aggregation trace conditioned on those traces; all components are optimized end-to-end against the reward of the final aggregated response. To train this system, SPIRAL uses set reinforcement learning to teach models to produce a set of traces that are collectively useful for an aggregator and standard reinforcement learning to teach models to aggregate the set into improved final responses. Our experiments on reasoning tasks show that SPIRAL effectively scales with inference compute, outperforming GRPO by up to 11$\times$ scaling efficiency and 15% higher performance when all three compute primitives are scaled.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前语言模型在后训练阶段仅针对单一顺序推理（链式思维）进行优化，忽视了测试时实际使用的并行推理和聚合推理这两种重要计算原语。这导致模型在需要利用额外推理计算时，无法有效地分配和协调计算资源，表现为顺序计算中思维预算错配、并行计算中探索冗余、聚合计算中无法合成异质解等瓶颈。
- **整体含义**：提出 SPIRAL（顺序‑并行‑聚合强化学习）框架，旨在训练模型端到端地使用顺序、并行和聚合三种推理计算原语，从而更有效地利用测试时计算资源，提升模型在复杂推理任务上的性能。

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：将推理管线建模为多层级强化学习问题：先并行采样一组搜索轨迹（顺序推理+并行计算），再基于这些轨迹生成聚合轨迹（聚合计算）。仅使用最终聚合响应的奖励来同时优化搜索轨迹的生成和聚合轨迹的生成。
- **关键技术细节**：
  - 搜索轨迹的优化采用**集合强化学习（Set RL）**：每组搜索轨迹作为一个整体（集合）获得共享奖励，奖励定义为该集合下聚合轨迹的平均奖励。利用边际集合优势（marginal set advantage）对每个搜索轨迹进行信用分配，鼓励模型生成集体有用的多样化轨迹。
  - 聚合轨迹的优化采用**标准强化学习**（类似GRPO）：每个聚合轨迹获得自身奖励减去组内基线后的优势，引导模型有效合成给定的搜索轨迹。
  - **算法流程**（Fig. 2）：
    1. 给定问题 x，采样 N1 个搜索轨迹 y1,…,yN1 ~ πθ(·|x)。
    2. 从这 N1 个轨迹中均匀无放回采样 K 个大小为 n 的集合 G1,…,GK。
    3. 对每个集合 Gi，采样 N2 个聚合轨迹 y^{Gi}_1,…,y^{Gi}_{N2} ~ πθ(·|x, Gi)。
    4. 计算每个聚合轨迹的奖励 r(x, y^{Gi}_j)。
    5. 聚合轨迹优势：A(x, y^{Gi}_j) = r(x, y^{Gi}_j) - (1/N2)Σ_k r(x, y^{Gi}_k)。
    6. 集合奖励：f̂_spiral(x, Gi) = (1/N2)Σ_j r(x, y^{Gi}_j)。
    7. 集合基线：f̂_spiral(x) = (1/K)Σ_i f̂_spiral(x, Gi)。
    8. 集合优势：A^♯(x, Gi) = f̂_spiral(x, Gi) - f̂_spiral(x)。
    9. 搜索轨迹 y_j 的边际集合优势：A^♯_marg(x, y_j) = 平均包含 y_j 的集合优势。
    10. 将上述优势代入策略梯度更新，总梯度 = 集合RL梯度（搜索轨迹）+ 标准RL梯度（聚合轨迹）。
- **理论保证**：证明了即使集合目标函数非对称，在从有序元组采样集合的设定下，该梯度估计仍是无偏的（Proposition 3.1）。

### 6. 论文的主要结论与发现

- SPIRAL 能更有效地利用测试时推理计算，在 pass@k 上相比 GRPO 获得**最高 11 倍的扩展效率**（即同等并行尝试数下覆盖率提升更显著）。
- 在递归自聚合设置下，SPIRAL 比 GRPO 和基础模型**高出最多 13.5% 的准确率**（Fig.4）。
- 在顺序计算扩展上，SPIRAL 与 GRPO 性能相近；但并行和聚合计算扩展才是 SPIRAL 的优势所在。
- 模型基于搜索轨迹的聚合能力（SPIRAL）优于基于规则的聚合（majority voting）。
- SPIRAL 的搜索轨迹多样性更高（token 级熵不坍缩），有助于并行探索。

### 7. 优点

- **端到端联合优化**：同时训练搜索和聚合，使其协同进化，而非分开设计。
- **集合强化学习的创新应用**：将搜索轨迹的集体效用作为优化目标，鼓励生成多样化且互补的轨迹。
- **理论保证**：给出了梯度估计的无偏性证明，保证了算法的合理性。
- **扩展效率显著**：在同等或更少 token 预算下，SPIRAL 的性能超越 GRPO 和基线。
- **兼容现有测试时扩展方法**：可以直接用于递归自聚合等复杂管线。

### 8. 不足与局限

- **训练规模较小**：仅在 4B 参数模型上实验，未验证更大模型（如 8B 或更大）的表现。
- **任务单一**：仅测试了数学推理，未扩展到其他领域（如代码、科学发现、通用问答）。
- **集合大小固定**：训练时固定 n=4，而测试时可能使用不同大小；虽然理论上可扩展，但缺乏实验验证。
- **可能存在的排序偏差**：语言模型对输入顺序敏感，而 SPIRAL 假设目标函数对称（使用无序集合），文中虽给出理论修正但实践中可能仍有影响。
- **未独立评估聚合和搜索能力**：改进来源（更好的搜索 vs 更好的聚合）未完全分离，缺少消融实验（如单独训练搜索或聚合）。
- **对比方法的局限性**：GRPO 本身也未针对聚合训练，但 SPIRAL 在并行和聚合扩展上的优势可能是由于训练时使用了更多 token（虽然总量相等但分配方式不同），文中已有控制，但潜在收益可能来自训练时的“曝光”。
