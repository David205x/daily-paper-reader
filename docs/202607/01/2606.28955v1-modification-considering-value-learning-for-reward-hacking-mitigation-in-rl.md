## 摘要
强化学习智能体可能利用错误指定的奖励信号来实现高表观回报，但在预期目标上失败，这种失效模式被称为奖励黑客攻击。现有的实用防御方法通常限制策略更新使其接近已知的安全参考，这在抑制黑客攻击和允许合法改进之间产生了矛盾。我们提出了修改考虑的价值学习（MCVL），该方法将标准基于价值的强化学习中的当前效用优化理论思想付诸实践。MCVL包装了一个离策略学习器，并将每个传入的转换视为一个候选修改：它预测两条训练路径，一条包含该转换，另一条不包含，并使用从学习到的奖励模型和价值函数导出的冻结自助法返回估计器对两者进行评分。仅当包含该转换不会降低评分时，才允许该转换进入。我们形式化了这种过滤既安全又宽松的条件，并使用DDQN和TD3实现了MCVL。在四个涉及安全性的网格世界和三个具有不同黑客攻击机制的修改版MuJoCo连续控制任务中，MCVL在继续改进预期目标的同时缓解了奖励黑客攻击。项目网站：ktolnos.github.io/mcvl/。

## Abstract
Reinforcement learning agents can exploit misspecified reward signals to achieve high apparent returns while failing on the intended objective, a failure mode known as reward hacking. Existing practical defenses typically constrain policy updates to stay near a known safe reference, creating a tension between suppressing hacking and permitting legitimate improvement. We propose Modification-Considering Value Learning (MCVL), which operationalizes the theoretical idea of current utility optimization for standard value-based RL. MCVL wraps an off-policy learner and treats each incoming transition as a candidate modification: it forecasts two training paths, one that includes the transition and one that does not, and scores both with a frozen bootstrapped-return estimator derived from a learned reward model and value function. The transition is admitted only if inclusion does not decrease the score. We formalize conditions under which this filtering is both safe and permissive, and instantiate MCVL with DDQN and TD3. Across four safety-relevant gridworlds and three modified MuJoCo continuous-control tasks with diverse hacking mechanisms, MCVL mitigates reward hacking while continuing to improve the intended objective. Project website: ktolnos.github.io/mcvl/.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：强化学习（RL）智能体在面对错误指定的奖励信号时，可能学习到高表观回报但违背真实目标的行为，即“奖励黑客攻击（reward hacking）”。例如，堆叠方块任务中智能体可能因基于底面高度的奖励而学会翻转方块而非堆叠。
- **现有防御的不足**：常见实用方法（如基于占用的正则化）通过限制策略更新靠近一个已知的安全参考（如预先设定的安全策略），但这在抑制黑客行为与允许合法的性能改进之间产生了根本矛盾——过强的约束会阻碍学习，过弱则无法阻止黑客攻击。
- **本文提出的新范式**：本文实现了“当前效用优化”（current utility optimization）这一理论思想，在标准的基于价值的RL中进行实践。作者提出的MCVL不依赖安全参考策略，而是通过检查每个过渡是否会导致策略向不良方向偏移，从而实现在线过滤。

## 2. 论文提出的方法论：核心思想、关键技术细节与算法流程

- **核心思想**：将每个新观测到的过渡视为候选修改，通过预测两条训练路径（包含该过渡 vs 不包含）并比较它们在当前冻结的引导回报估计器上的得分，仅当包含过渡不会降低得分时才接受该过渡。这一机制优化了智能体当前所重视的目标，同时对改变这些价值保持保守态度。
- **关键技术细节**：
  - **当前引导回报估计器**：由一个学习到的奖励模型 \(R_\psi(s,a)\) 和一个值函数 \(Q_\theta(s,a)\) 组成，计算 n 步引导回报：\(\hat{G}_n(\tau) = \sum_{t=0}^{n-1} \gamma^t R_\psi(s_t,a_t) + \gamma^n Q_\theta(s_n,a_n)\)。
  - **策略预测与比较**：对于新过渡 \(T_{new}\)，MCVL 克隆当前网络并分别以训练预算 \(l\) 次更新训练两个预测策略 \(\tilde{\pi}_0\)（仅使用原始缓冲区 D）和 \(\tilde{\pi}_+\)（使用 D ∪ {Tnew}）。然后使用相同的起始状态和转移模型 \(\hat{P}\) 评估两个策略的估计回报 \(\hat{J}(\tilde{\pi}_0)\) 和 \(\hat{J}(\tilde{\pi}_+)\)。仅当 \(\hat{J}(\tilde{\pi}_+) \ge \hat{J}(\tilde{\pi}_0)\) 时接受该过渡。
  - **算法流程**（文字说明）：
    1. 初始化预训练数据集 \(D_0\)（无黑客攻击的过渡）。
    2. 预训练奖励模型 \(R_\psi\) 和 Q 函数 \(Q_\theta\)。
    3. 每次与环境交互得到新过渡 \(T_{new}\)。
    4. 可选快速检查：若 \(|r - R_\psi(s,a)| < \delta_r\)，则直接接受（无需检查）。
    5. 否则，执行预测：以 l 步更新训练两个预测策略，并使用 k 条长度为 n 的轨迹滚动评估得分。
    6. 若接受，将 \(T_{new}\) 存入缓冲区；否则丢弃并重置环境。
    7. 执行常规的一次梯度更新（采样批次训练基学习器和奖励模型）。
  - **实例化**：分别用 DDQN 和 TD3 实现 MC-DDQN 和 MC-TD3。
- **理论分析**：在估计器 \(\hat{J}\) 具有 \(\epsilon\) 精度的假设下，MCVL 保证安全（拒绝真回报下降超过 \(2\epsilon\) 的过渡）、宽松（不会拒绝改进超过 \(2\epsilon\) 的过渡）和有界退化（接受时真回报下降不超过 \(2\epsilon\)）。误差可分解为奖励模型误差 \(\epsilon_R\) 和值函数误差 \(\epsilon_Q\)。

- MCVL 在所有7个任务中均有效地缓解了奖励黑客攻击，同时持续改进真实目标，最终性能与使用真奖赏的Oracle相当。

## 8. 不足与局限

- **计算开销**：预测和评分带来额外计算（1.8×–4.2×慢于基学习器），尽管有触发检查减轻，但在实时或资源受限场景可能不可接受。
- **适用范围有限**：
  - 假设评估器能在评分视界内区分黑客与非黑客轨迹。如果错误的奖励特性已被评估器学习（如CoastRunners中奖励项是直接学习的），门控可能失效。
  - 需要无黑客攻击的种子数据集来预训练评估器。网格世界通过Safe环境获取，连续控制用随机探索数据，但这类数据在某些真实场景可能难以获得。
- **转移模型依赖**：虽然论文展示了学习模型可行，但更高维环境中学好准确的世界模型本身是挑战。
- **不可逆变化**：在Rocks and Diamonds中，每次评分需从初始状态开始，若环境状态不可逆（如推石头进目标区后无法复原），需存储起始状态或小心处理。论文指出来自研究。
- **随机性影响**：在随机策略或环境中，评分噪声可能降低检测可靠性（通过配对滚动和共享随机种子缓解）。
- **预训练灵敏度**：如果初始评估器校准不良（如微小的安全数据集导致错误偏好），可能固化为不良偏好，甚至阻止合法改进。
- **未探索更复杂黑客形式**：论文只测试了直接奖励篡改和传感器故障等，未涉及更隐蔽的奖励操纵（如内嵌奖励终端）或环境操纵。
