## 摘要
将独立训练的LoRA适配器组合成一个大型语言模型对于多领域适应很有用，特别是当原始训练数据无法共享时。一种常见的方法是对LoRA专家使用MoE风格的路由，但对于冻结的预训练适配器，软加权组合可能会改变每个LoRA模块最初训练时的单位尺度加性更新。我们提出Hard-Routed MoR-LoRA，一个两阶段框架，通过单位尺度硬选择来组合冻结的推理LoRA专家。首先，使用来自可验证反馈的强化学习独立训练领域特定的LoRA适配器，以获得推理专家。然后，冻结所有专家，从中蒸馏出推理轨迹，仅训练一个轻量级共享路由器以及一个小型注意力LoRA用于集成。路由器使用硬top-1路由为每个令牌精确选择一个专家，同时通过直通估计器实现基于梯度的训练。在五个基准、多个模型规模和额外模型系列上的实验表明，Hard-Routed MoR-LoRA保留了专家行为，同时所需可训练参数比软路由混合基线少得多。我们的进一步分析表明，归一化软混合通常将大部分路由质量集中在一个专家上，这表明硬单位尺度路由为冻结的LoRA专家组合提供了一种简单高效的抽象。

## Abstract
Composing independently trained LoRA adapters into a single large language model is useful for multi-domain adaptation, especially when the original training data cannot be shared. A common approach is to use MoE-style routing over LoRA experts, but for frozen pretrained adapters, soft weighted combinations can change the unit-scale additive update under which each LoRA module was originally trained. We propose \textbf{Hard-Routed MoR-LoRA}, a two-stage framework for composing frozen reasoning LoRA experts through unit-scale hard selection. First, domain-specific LoRA adapters are trained independently using reinforcement learning from verifiable feedback to obtain reasoning experts. Then, all experts are frozen, reasoning traces are distilled from them, and only a lightweight shared router together with a small attention LoRA is trained for integration. The router selects exactly one expert per token using hard top-1 routing, while a straight-through estimator enables gradient-based training. Experiments across five benchmarks, multiple model scales, and additional model families show that Hard-Routed MoR-LoRA preserves expert behavior while requiring substantially fewer trainable parameters than soft-routing mixture baselines. Our analysis further shows that normalized soft mixtures often concentrate most routing mass on a single expert, suggesting that hard unit-scale routing provides a simple and efficient abstraction for frozen LoRA expert composition.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：将多个独立训练的 LoRA 适配器组合到同一个大语言模型中，是实现多领域适应的常见需求（尤其在数据无法共享时）。现有方法大多采用 Mixture-of-Experts（MoE）风格的软路由（soft routing），对各个专家输出进行加权求和。然而，每个 LoRA 模块在其原始训练中假设以单位尺度（unit-scale）进行加性更新；软路由的加权组合会改变实际应用的更新量，造成“尺度不匹配（scale mismatch）”，从而破坏专家原有行为。
- **动机**：为了避免重新训练或修改专家参数，需要一种能够保留原始 LoRA 更新的集成策略。论文提出将集成视为一个“选择问题”而非“再学习问题”，通过对冻结专家进行硬路由（hard routing）来确保单位尺度应用。

## 2. 方法论

- **核心思想**：两阶段框架，解耦推理能力获取与专家选择。阶段 I 使用基于可验证反馈的强化学习（RLVF）独立训练领域专用的推理 LoRA 专家。阶段 II 冻结所有专家，仅训练一个轻量级共享路由器和一个小型注意力 LoRA，实现硬 top-1 路由选择。
- **关键技术细节**：
  - **硬路由与直通估计器（STE）**：路由器输出 softmax 概率，但前向传播执行离散的 argmax 选择，只激活一个专家并应用单位尺度输出（即 \( E_j(x) \)）。反向传播时使用直通估计器，将梯度通过 soft 概率回传，从而保持可微性。
  - **共享路由器**：一个线性层，对所有层和令牌共享，减少参数量。
  - **注意力 LoRA**：在集成阶段对注意力投影（q/k/v/o）添加低秩适配，以协调令牌级路由可能带来的表示变化。

## 6. 主要结论与发现

1. **RLVF 产生更强的推理专家**：对于具有足够推理能力的中大型模型（3B/8B），RLVF 训练比 SFT 能更好地提升推理性能，且不会破坏模型已有的 CoT 行为；而普通微调反而可能降低能力。
2. **硬路由在冻结专家设置下优于软路由**：Hard-Routed MoR-LoRA 在保持专家行为的同时，可训练参数远少于 LoRAMixer 等软路由方法（如 LLaMA-3B 下 73M vs 606M），且平均准确率更高或相当。
3. **归一化软路由隐式接近硬选择**：分析发现归一化 top-2 路由中，平均主导权重约 0.71，路由熵较低，说明其实际行为几乎等同于单专家选择；硬路由提供了更直接、更高效的实现。

## 7. 优点

- **方法简洁有效**：通过直通估计器实现离散硬路由，无需重新训练专家，保持参数高效。
- **解耦学习**：将推理能力获取与路由学习分开，阶段 II 只需少量监督数据（每领域 1000 条）即可学习选择策略。
- **跨模型泛化**：在 LLaMA、Gemma、Qwen 上均验证有效，说明方法通用。
- **实验充分**：进行了大量消融和对比实验，支撑结论。

## 8. 不足与局限

- **仍需少量标注数据**：阶段 II 不是零样本集成，需要每个领域至少几百到一千条提示来训练路由器，限制了完全数据不可共享场景的直接适用。
