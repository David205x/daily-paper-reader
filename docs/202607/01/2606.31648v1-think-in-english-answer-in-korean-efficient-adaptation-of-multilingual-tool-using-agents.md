## 摘要
我们提出了LuckyStar 111B，一个111B参数的混合推理模型，由Cohere和LG CNS合作开发，用于在实际内存和服务限制下的韩英企业代理。该模型从Cohere完全后训练的Command A模型出发，而非从头预训练，并通过前置条件调节在简洁的非推理行为与更长的面向工具推理之间切换。我们研究了高效扩展工具使用代理的四种选择：多语言监督微调、基于可验证奖励的多步工具使用任务强化学习、面向韩语用户的语言一致性奖励，以及用于单GPU服务的4位量化。适配后的模型在数学推理、函数调用和代理式自然语言到SQL（NL2SQL）性能上有所提升，同时保持了韩语和英语通用指令遵循的质量。这些结果为在内存受限部署下将后训练多语言模型适配到可验证的代理工作流程提供了实用方案和失败模式分析。

## Abstract
We present LuckyStar 111B, a 111B-parameter hybrid reasoning model developed through a collaboration between Cohere and LG CNS for Korean-English enterprise agents under practical memory and serving constraints. The model trains from Cohere's fully post-trained Command A model rather than a new pretraining run, and uses preamble conditioning to switch between concise non-reasoning behavior and longer tool-oriented reasoning. We study four choices for scaling tool-using agents efficiently: multilingual supervised fine-tuning, reinforcement learning with verifiable rewards for multi-step tool-use tasks, language-consistency rewards for Korean user-facing responses, and 4-bit quantization for single-GPU serving. The adapted model improves mathematical reasoning, function calling, and agentic natural-language-to-SQL (NL2SQL) performance while preserving general Korean and English instruction-following quality. These results provide a practical recipe and failure-mode analysis for adapting post-trained multilingual models to verifiable agentic workflows under memory-constrained deployment.

- **核心问题**：如何在不重新预训练、不增加部署成本的前提下，高效地将一个已后训练的多语言基础模型适配为支持工具调用的双语（韩英）企业级代理，同时满足内存和推理延迟的严格限制。
- **背景与动机**：
  - 企业级助手常需处理私有数据、调用工具（如SQL、检索），并以目标语言（如韩语）返回答案。
  - 现有模型在数学推理、函数调用和NL2SQL等可验证任务上表现不足。
  - 多语言场景下存在“推理语言漂移”问题：韩语提示词倾向于生成英语最终答案。
  - 实际部署要求模型压缩（如4-bit量化）以在单GPU上运行。
- **整体意义**：提供一套从后训练模型出发、结合混合推理、可验证强化学习、语言一致性惩罚和量化压缩的实用配方，并分析了关键失败模式，为内存受限下的多语言代理适配提供参考。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：采用**前置条件（preamble）调节**实现混合推理模式 – 同一模型权重通过切换prompt前缀来改变行为（推理模式 vs 简洁模式）。
- **三阶段适配流水线**：
  1. **混合监督微调（Hybrid SFT）**：80%推理数据（数学、代码、工具使用） + 20%非推理指令遵循数据。推理数据使用“推理前置条件”，非推理数据使用“简洁前置条件”。通过拒绝采样生成正确推理轨迹（仅保留精确匹配答案的轨迹）。
  2. **基于可验证奖励的强化学习（RLVR）**：使用REINFORCE Leave-One-Out (RLOO)算法，对数学和NL2SQL任务设定二进制奖励。关键创新：
     - 对韩语提示增加**语言一致性惩罚**：\( r = r_{\text{correctness}} - r_{\text{penalty}} \)，当最终答案语言与提示语言不匹配时惩罚0.5。
     - 过滤超过32k token的 rollout，避免过早终止。
  3. **离线偏好对齐（DPO）**：使用低学习率进行直接偏好优化，纠正RLVR引入的冗长问题。偏好对包括来自基础模型对齐管道的数据和新构建的韩语特定偏好对，以及针对简洁性的目标偏好对（直接答案 vs 冗长输出）。
- **多语言推理策略**：采用“用英语思考，用目标语言回答”策略。对2万条韩语中等难度提示，用英语生成推理轨迹+韩语最终答案，并通过拒绝采样验证后加入SFT。
- **NL2SQL数据构建**：从Spider、BIRD、SynSQL筛选10万条可执行且输出长度1-500 token的SQL查询。基础模型准确率<1%导致冷启动问题，通过best-of-N拒绝采样生成正确轨迹，再用SFT引导后进入RLVR。
