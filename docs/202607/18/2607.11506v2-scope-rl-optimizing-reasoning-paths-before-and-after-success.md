## 摘要
基于可验证奖励的强化学习（RLVR）使用稀疏的可验证最终答案奖励来优化大语言模型。这种稀疏锚点可靠地验证轨迹是否成功，但不会对其产生的推理路径提供直接反馈。在成功之前，困难问题上的先决条件进展得不到奖励信号；在成功之后，结果奖励无法区分组织良好的正确轨迹与冗余或局部有缺陷的轨迹。我们提出了SCOPE-RL（带过程效率的支架式链式优化），这是一个两阶段框架，它在保留GRPO更新的同时使该锚点变得密集：自适应支架式强化学习在成功之前，在隐藏答案的子问题链上添加前缀分解的可验证奖励；成功之后，质量感知过程强化学习应用基于正确性门控的过程形状奖励来优化正确的轨迹。一个经过专家验证的步骤质量评估协议评估有效步骤密度、错误定位和令牌效率，超越了最终答案的准确性。在DAPO-Math和Big-Math上训练的Qwen3-8B-Instruct上，SCOPE-RL相比仅使用结果的GRPO平均准确率提高了多达11.2个百分点，推理令牌减少了多达27.1%；这些提升在GSPO和Qwen3-0.6B-Instruct上仍然保持，表明奖励信号密集化与策略更新层面的RLVR进展互补。代码和数据见https://github.com/tokencraft-lab/SCOPE-RL。

## Abstract
Reinforcement learning with verifiable rewards (RLVR) optimizes LLMs using sparse verifiable final-answer rewards. This sparse anchor reliably verifies whether a trajectory succeeds but provides no direct feedback on the reasoning path that produced it. Before success, prerequisite progress on hard problems receives no reward signal; after success, outcome rewards cannot distinguish well-organized correct trajectories from redundant or locally flawed ones. We introduce SCOPE-RL (Scaffolded Chain Optimization with Process Efficiency), a two-stage framework that densifies this anchor while retaining the GRPO update: Adaptive Scaffolded RL adds prefix-decomposed verifiable rewards on answer-hidden sub-question chains before success, and Quality-Aware Process RL applies correctness-gated process-shape rewards to refine correct trajectories after success. An expert-validated Step-Quality Evaluation Protocol evaluates useful-step density, error localization, and token efficiency beyond final-answer accuracy. On Qwen3-8B-Instruct trained on DAPO-Math and Big-Math, SCOPE-RL improves average accuracy by up to 11.2 pp and reduces reasoning tokens by up to 27.1% over outcome-only GRPO; the gains hold under GSPO and on Qwen3-0.6B-Instruct, indicating that reward-signal densification is complementary to policy-update-level RLVR advances. Code and data are available at https://github.com/tokencraft-lab/SCOPE-RL.

# SCOPE-RL: 在成功前后优化推理路径 — 详细中文总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

基于可验证奖励的强化学习（RLVR）使用**稀疏的最终答案奖励**（如正确/错误）来优化大语言模型的推理能力。这种“稀疏锚点”虽然能可靠地判断轨迹是否成功，但不能对推理过程提供直接反馈，导致两个失效模式：
- **成功前（Before Success）**：对于困难问题，模型很少能产生正确的最终答案，因此几乎得不到奖励信号，训练停滞。
- **成功后（After Success）**：即使模型最终答对，端点奖励也无法区分是清晰高效的推理还是冗余、局部有缺陷的推理，导致模型倾向于生成冗长而低质量的轨迹。

现有工作分别从探索、效率、过程监督等角度应对这两个问题，但较少研究如何通过**奖励信号本身稠密化**来同时解决成功前和成功后的监督不足。该论文提出**SCOPE-RL**，在保持GRPO更新不变的前提下，通过在成功前添加可验证的子目标奖励、成功后加入正确性门控的过程质量奖励，来稠密化稀疏锚点。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 核心思想
SCOPE-RL是一个**两阶段顺序训练框架**：
- **Stage 1: 自适应支架式强化学习（Adaptive Scaffolded RL, ASR）** — 针对成功前阶段，将困难问题分解为隐藏答案的子问题链，对每个子答案提供前缀一致的可验证奖励，从而在最终答案之前提供稠密的前置进度信号。
- **Stage 2: 质量感知过程强化学习（Quality-Aware Process RL, QPR）** — 针对成功后阶段，对已经答对的轨迹，通过一个固定的LLM Judge（不与策略同步训练）为每个推理步骤标注类别（有用、机械、冗余、回退、错误），并组合成过程形状奖励，优化正确轨迹的紧凑性和有用步骤密度。

两阶段顺序耦合：Stage 1 先提升低难度问题的成功率和轨迹供应，Stage 2 再对这些正确轨迹进行精炼，避免混合训练时过程惩罚过早施加。

### 关键技术细节

#### Stage 1: 自适应支架式RL（ASR）
- **离线分解**：使用教师模型将问题分解为子问题链 `(q1, a1), ..., (qm, am), (q_main, a)`，并保证子答案对模型隐藏、可自动验证、依赖关系正确。
- **在线路由**：每个batch先用原始提示词采样并计算平均结果奖励 `r̄(q)`。若 `r̄(q) < τ`（实验中τ=0.5），则路由到支架提示词，重新采样并用支架奖励。
- **前缀一致奖励**：
  ```
  RASR = β * (1/m) * Σ Π_i + (1-β) * Π_m * 1[答案正确]
  ```
  其中 `Π_i = ∏_{j=1}^i 1[ˆa_j = a_j]` 表示前i个子答案全部正确，保证了只有按顺序正确完成子问题才能获得奖励，防止跳过前置步骤。
- 混合奖励 `R1`：路由组用 `RASR`，非路由组用结果奖励 `r_out`。

#### Stage 2: 质量感知过程RL（QPR）
- **正确性门控**：只对最终答案正确的轨迹施加过程形状奖励，错误轨迹奖励为零。
- **步骤标注**：使用固定LLM Judge（GPT-4.1-mini）将轨迹解析为原子步骤，并为每步分配五类标签：有用、机械、冗余、回退、错误。
- **奖励组合**：
  - 有用步骤比例 `S_u` （正信号）
  - 低价值惩罚因子 `Φ = ∏_{k∈{mec,red,rev,err}} (1 - λ_k * r_k)` （每个类别惩罚系数小于1，保证正性）
  - 紧凑性因子 `κ = 1 / (1 + α ln(N+1))` （软性长度惩罚，不鼓励无限制增长）
  - 最终 `Q_process = S_u * Φ * κ`，范围[0,1]。
- QPR优化时不修改提示词格式，模型在推理时不再需要显式子问题结构。

### 算法流程（文字描述）

1. **Stage 1**：对每个批次的提示词，用当前策略在原始提示下采样G=8条轨迹，计算平均结果奖励。若低于阈值τ，则路由到缓存好的支架提示词，重新采样并用`R_ASR`评分；否则保留原始轨迹和结果奖励。合并所有组，用GRPO更新策略。
2. **Stage 2**：从Stage 1最佳检查点初始化。对每个原始提示词采样G=8条轨迹，只对正确轨迹调用LLM Judge获取步骤类别，计算`Q_process`作为奖励。错误轨迹奖励为0。用GRPO更新策略。

## 6. 论文的主要结论与发现

1. **准确率显著提升**：在DAPO-Math上，SCOPE-RL平均准确率55.80% → 66.35%（+11.2pp）；在Big-Math上53.58% → 64.79%（+11.2pp）。最大收益来自AIME等高难度基准。
2. **推理效率提高**：平均推理token减少16.2%（DAPO-Math）和27.1%（Big-Math），且有用步骤比例上升，说明减少的是冗余和错误步骤而非必要推理。
3. **过程质量改善**：有用步骤比例（Useful%）持续上升，首个错误位置后移，表明推理更加集中和可靠。
4. **奖励稠密化与策略更新方法互补**：在GSPO后端和Qwen3-0.6B等不同设置下增益保持，说明奖励信号改进可以与更高级的策略更新方法叠加。
5. **支架前缀的正确性可预测主答案的正确性**：实验证明，子问题前缀正确率与最终答案准确率单调相关（图1a），这为ASR的设计提供了经验依据。
6. **正确性门控是过程奖励有效的关键**：QPR的关键在于只在正确轨迹上施加过程奖励；若混合奖励或直接使用简单压缩提示，会损失准确率或质量。

## 7. 优点

- **奖励层面稠密化，不改变策略更新**：方法直接作用于奖励设计，与任何基于GRPO的优化器兼容，易于集成到现有RLVR框架。
- **两阶段分离逻辑清晰**：先提升成功率（ASR），再优化正确轨迹质量（QPR），避免了联合优化时的信号冲突。
- **子问题支架隐藏答案并保证非泄露**：通过严格过滤谓词（Eq.(6)）确保模型无法从提示词中窃取答案，保持了“可验证奖励”的纯洁性。
- **正确性门控的QPR设计**：防止了过程奖励鼓励短而错的推理，同时通过固定LLM Judge避免了奖励与策略共适应，保证了过程的客观性。
- **广泛的实验验证**：跨数据源、跨优化器、跨模型规模、跨评估基准，且有专家验证的步骤质量协议，结果可信度高。
- **步骤质量评估协议本身有贡献**：提供了一个不依赖训练的可推广的诊断工具，可用于未来RLVR研究。

## 8. 不足与局限

- **模型范围有限**：实验仅限于Qwen系列（8B和0.6B），未在更大模型（如70B）或其他模型族（如Llama、DeepSeek）上验证。
- **领域限制**：仅测试数学和科学推理（GPQA、MATH、AIME），未覆盖代码生成、符号推理或其他可验证奖励领域。
- **方法对比不全面**：未与所有最新的探索式或效率型RLVR方法（如EvoCoT、StepHint、Prime等）进行直接比较；作者将此归为有意的范围选择，但降低了比较的完备性。
- **超参数调优有限**：仅对路由阈值τ进行了敏感性分析（附录K），其他超参数（β、α、各λ_k）未进行系统性搜索；不同问题分布可能需要不同的惩罚权重。
- **支架分解依赖教师模型**：ASR需要先离线生成子问题链，该过程可能引入教师模型的能力偏差或遗漏关键分解路径；虽然做了人工验证，但仍有限制。
- **QPR中LLM Judge的偏见**：虽然使用不同模型（GPT-4.1-mini）进行训练标注，但仍可能包含LLM的系统性偏好，且标注质量未在所有数据上人工验证（仅验证了有用/无用二元区分）。
- **Pass@128曲线有瑕疵**：在AIME 2025-I上，SCOPE-RL的pass@128略低于GRPO（80% vs 86.67%），说明质量精炼可能牺牲了高采样预算下的覆盖度，对于需要极致覆盖的场景可能不是最优选择。
