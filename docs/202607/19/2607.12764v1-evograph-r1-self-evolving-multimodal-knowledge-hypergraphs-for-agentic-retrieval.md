## 摘要
检索增强生成（RAG）已成为将多模态大语言模型（MLLM）锚定于外部知识的关键范式。最近的GraphRAG方法引入了结构化实体关系图来改进检索和推理，但仍受限于将知识图谱视为离线构建且一次性查询的静态数据结构。这种静态范式与知识密集型推理的交互式、迭代式特性不匹配，造成三个瓶颈：（i）以文本为中心的分裂阻碍跨模态推理，（ii）冻结的结构无法纳入新证据或纠正错误，（iii）刚性的一次性检索缺乏自适应优化。为克服这些局限，我们提出EvoGraph-R1，一种自演化GraphRAG框架，将知识图谱重新定义为通过智能体交互塑造的动态环境。我们将检索建模为马尔可夫决策过程（MDP），其中智能体观测图状态并执行动作：查询（GraphRetrieve）、扩展（WebSearch）、精炼（GraphEdit）或终止（Answer）推理。这些动作重塑超图结构并生成反馈信号，引导后续演化。通过这种闭环，超图通过整合新证据、纠正错误和精炼结构来支持多跳推理。在多模态VQA和纯文本QA基准上的实验表明，该方法在准确性、覆盖率和可追溯性上显著优于现有RAG基线，将自演化知识图谱确立为跨模态的基础范式。

## Abstract
Retrieval-augmented generation (RAG) has emerged as a critical paradigm for grounding Multimodal Large Language Models (MLLMs) in external knowledge. Recent GraphRAG methods introduce structured entity-relation graphs to improve retrieval and reasoning. However, they remain limited by treating knowledge graphs as static data structures built offline and queried in a single pass. This static paradigm misaligns with the interactive, iterative nature of knowledge-intensive reasoning, creating three bottlenecks: (i) text-centric fragmentation that impedes cross-modal reasoning, (ii) frozen structures unable to incorporate new evidence or correct errors, and (iii) rigid single-pass retrieval without adaptive refinement. To overcome these limitations, we introduce EvoGraph-R1, a self-evolving GraphRAG framework that reconceptualizes knowledge graphs as dynamic environments shaped through agent interactions. We formulate retrieval as a Markov Decision Process (MDP) where the agent observes the graph state and executes actions to query (GraphRetrieve), expand (WebSearch), refine (GraphEdit), or terminate (Answer) the reasoning. These actions reshape the hypergraph structure and generate feedback signals that guide subsequent evolution. Through this closed loop, the hypergraph evolves by integrating new evidence, correcting errors, and refining structure to support multi-hop reasoning. Experiments on multimodal VQA and text QA benchmarks demonstrate substantial improvements over existing RAG baselines in accuracy, coverage, and traceability, establishing self-evolving knowledge graphs as a fundamental paradigm across modalities.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：现有检索增强生成（RAG）方法，特别是GraphRAG，将知识图谱视为**静态数据结构**——离线构建、一次性查询，无法在推理过程中动态更新。这种静态范式与人类推理的**交互式、迭代式**特性严重不匹配，导致三个关键瓶颈：
  - **(i) 文本中心碎片化**：LLM提取将多模态证据塌缩为孤立文本元组，丢失跨模态依赖和长程关系。
  - **(ii) 静态结构**：图构建后固定不变，无法融入新证据、纠正错误或覆盖未见主题。
  - **(iii) 刚性单次检索**：检索缺乏自适应优化，初始证据不足时无法修正策略或调用外部搜索。
- **整体含义**：为解决这些问题，论文提出**EvoGraph-R1**，将知识图谱重新定义为**动态环境**，通过智能体（agent）与环境的交互实现**自演化**。这是首个将多模态GraphRAG建模为马尔可夫决策过程（MDP）的工作，实现了检索、推理与知识演化的闭环统一。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 将多模态GraphRAG形式化为**马尔可夫决策过程（MDP）**：
  - **环境状态**：多模态知识超图 \(G_t = (V_t, E_t)\)，包含实体、关系、视觉对象及其结构组织。
  - **动作空间**：四种离散动作：
    - `GraphRetrieve`：查询当前超图中的实体/子图。
    - `WebSearch`：当图中证据不足时，从外部搜索引擎获取信息。
    - `GraphEdit`：包含三个子操作：
      - `Insert`：插入新实体/超边。
      - `Update`：修改现有超边以纠正错误或解决冲突。
      - `Delete`：软删除（降低置信度）低质量或矛盾元素。
    - `Answer`：终止推理并基于当前图状态生成最终回答。
  - **奖励函数**：三种信号组合：
    - **结构奖励** \(R_{\text{struct}}\)：惩罚/奖励推理步骤的协议合规性（格式、结构）。
    - **答案奖励** \(R_{\text{ans}}\)：基于F1式词元重叠衡量预测与真实答案的语义对齐。
    - **整体奖励** \(R(\tau)\)：结合动作成本、结构奖励与答案奖励，平衡准确性与效率。
  - **策略**：通过强化学习（GRPO）学习智能体的动作选择策略 \(\pi_\theta(a_t|s_t)\)，以最大化期望奖励。

### 关键技术细节
1. **多模态超图构建**：
   - **文本子图**：用MLLM提取n元关系超边（包含描述、实体集、关系类型、置信度）。
   - **视觉子图**：对每张图像生成场景描述，提取实体和关系，并用锚节点（anchor node）保持图像上下文归属。
   - **跨模态融合**：通过实体解析（entity resolution）合并匹配节点，统一文本和视觉子图。
   - **离线索引**：用多模态编码器（GME）嵌入所有图元素，支持高效检索。

2. **智能体驱动的图演化**：
   - 初始化：从最小超图开始。
   - 每一步：智能体观察当前状态 \(s_t = (G_t, H_t, q)\)（图、历史、问题），选择动作。
   - 动态转换：`GraphRetrieve`仅标记已访问元素；`WebSearch`引入外部证据并更新上下文；`GraphEdit`直接修改图结构（插入/更新/删除）。
   - 历史记录：每一步的动作、奖励、新图状态追加到轨迹 \(H_t\) 中。
   - 终止：选择`Answer`动作或达到最大步数，基于最终图状态 \(G_T\) 生成答案。

3. **强化学习优化**：
   - 使用**GRPO**（Group Relative Policy Optimization），对每个问题采样多条轨迹，计算组内相对优势。
   - 奖励函数 \(R(\tau) = -\lambda \sum c(a_t) + R_{\text{struct}}(\tau) + \mathbb{I}[R_{\text{struct}}=1.0] \cdot R_{\text{ans}}\)，鼓励结构正确且答案准确的轨迹，同时惩罚高成本动作。

### 算法流程（文字说明）
1. 输入：问题 \(q\)，多模态文档集 \(B = \{(T_i, I_i)\}\)。
2. 构建初始超图 \(G_0\)（文本+视觉提取、跨模态融合、索引）。
3. 循环（步数 \(t=1\) 至 \(T\)）：
   - 智能体观测状态 \(s_t = (G_t, H_t, q)\)。
   - 根据策略 \(\pi_\theta\) 选择动作 \(a_t\)。
   - 执行动作，更新图 \(G_{t+1} = \pi_{\text{evolve}}(a_t, G_t)\)，记录历史 \(H_{t+1}\)。
   - 若动作非`Answer`，继续；若为`Answer`或达到步数上限，退出循环。
4. 由最终图状态 \(G_T\) 生成答案 \(a^* = \pi_{\text{final\_resp}}(q, H_T)\)。
5. 在训练时，根据轨迹奖励使用GRPO更新策略参数 \(\theta\)。

## 6. 主要结论与发现

- **性能领先**：EvoGraph-R1在文本和跨模态基准上均优于所有基线。
  - 文本：比Graph-R1高+3.5%～+6.9% F1（2Wiki/HotpotQA/NQ）。
  - 多模态：比MMSearch-R1高+6.7%～+8.7%准确率（E-VQA/OK-VQA）；比MMKB-RAG高+7.7%。
- **效率提升**：检索轮数更少（2.57轮/1.65轮），响应更简洁（~1300 tokens），优于无编辑变体（~3.1轮/2850 tokens）和MMSearch-R1（~3.5轮/2200 tokens）。
- **组件贡献**：消融显示每个组件（超图、编辑操作、WebSearch）均贡献显著，其中INSERT最关键（移除后F1下降8.4%），WebSearch其次（下降9.6%）。

- **图质量提升**：精炼后图密度增加7.81%，聚类系数增加16.67%，语义相似度增加3.16%。
- **跨模态泛化**：自演化范式有效推广至纯文本任务，验证了框架的通用性。
