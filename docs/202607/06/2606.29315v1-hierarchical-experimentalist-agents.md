## 摘要
大型语言模型（LLM）越来越多地被用于在现实世界中采取行动并支持人类决策，然而大多数代理依赖于参数化知识、固定的训练后数据、检索或搜索。这种范式在新颖领域以及无法仅凭先验知识回答的复杂查询中会失效。例如，了解物理定律本身并不能使LLM回答复杂物理系统中的查询或完成长期任务。为解决这一问题，我们引入了分层实验代理（HExA），一种通过主动实验进行学习的上下文自改进框架。HExA迭代地设计和优化与查询相关的实验，从经验中学习可复用的可组合技能库，并整合实验证据以回答查询或采取行动。HExA无需训练，兼容任何黑盒模型，且不需要外部监督、预言机或离线数据。为评估主动实验，我们引入了Interphyre，一个基于PHYRE 2D程序化物理环境的工具调用基准，代理通过模拟API提出干预措施并测试假设。实验表明，当前的LLM代理在这些设置中表现不佳，尤其是在Interphyre最难级别上。Claude Sonnet 4.6仅达到2%的成功率，而HExA将同一模型提升至高达77%的成功率。HExA还改进了开放权重模型，并优于ReAct和Reflexion等代理基线。此外，仅使用从较容易级别学习并转移的技能（无需主动实验），HExA达到了44%的成功率，展示了其学习技能的可复用性和泛化能力。总体而言，HExA表明通过主动实验学习可以帮助代理发现有用知识、获取可复用技能，并在新颖的长期任务中取得高效进展。

## Abstract
Large language models (LLMs) are increasingly used to take actions in the real world and support human decision-making, yet most agents rely on parametric knowledge, fixed post-training data, retrieval, or search. This paradigm breaks down in novel domains and for sophisticated queries that cannot be answered from prior knowledge alone. Knowing the laws of physics, for instance, does not by itself enable LLMs to answer queries or complete long-horizon tasks in a complex physical system. To address this, we introduce Hierarchical Experimentalist Agents (HExA), an in-context self-improvement framework to learn from active experimentation. HExA iteratively designs and refines query-relevant experiments, learns a reusable library of composable skills from experience, and integrates experimental evidence to answer queries or take actions. HExA is training-free, compatible with any black-box model, and does not require external supervision, oracles, or offline data. To evaluate active experimentation, we introduce Interphyre, a tool-calling benchmark built on the PHYRE 2D procedural physics environment, where agents propose interventions and test hypotheses through simulation APIs. Experiments show that current LLM agents struggle in these settings, especially on the hardest levels of Interphyre. Claude Sonnet 4.6 achieves only 2% success, while HExA improves the same model to up to 77% success. HExA also improves open-weight models and outperforms agentic baselines such as ReAct and Reflexion. Moreover, using only skills learned from easier levels and transferred without active experimentation, HExA achieves 44% success, demonstrating the reusability and generalization of its learned skills. Overall, HExA shows that learning through active experimentation can help agents discover useful knowledge, acquire reusable skills, and make efficient progress on novel long-horizon tasks.

### 1. 核心问题与整体含义（研究动机和背景）

- **背景**：当前 LLM 代理主要依赖预训练的**参数化知识**（通过模仿学习或检索增强生成），这要求训练语料已经覆盖了相关领域。然而，对于**全新环境**（例如复杂的物理系统）或无法仅凭先验知识回答的**复杂查询**，这种方式会失效：即使模型知道物理定律，也无法直接用它们求解具体的实验问题。
- **核心问题**：如何让 LLM 代理**通过主动实验进行推理和行动**，即：探索环境、收集有针对性的数据、提出可证伪的假设、从中学习可复用的技能，并将这些技能迁移到相关任务中，从而高效地解决新颖的长期任务？
- **整体意义**：论文提出了一种**以实验为中心**的智能体范式，认为真正的智能应像科学家一样通过交互和实验学习，而非仅靠静态知识的回忆。该工作通过一个无需训练、全上下文学习的框架 HExA 实现了这一想法，并在物理推理测试床 Interphyre 上显著提升了性能。

### 2. 方法论：核心思想、关键技术细节

- **核心思想**：HExA 是一个**上下文强化学习框架**，它让 LLM 代理通过**主动实验**来获取查询特定的证据，从成功和失败的轨迹中**蒸馏出可复用的技能**（以自然语言形式存储在外部的技能库中），并在后续 episode 中**检索并注入**这些技能以指导实验，从而摊销探索成本。
- **关键技术细节**：
    - **三组件循环**（图2b）：
        - **Actor**（演员）：任意可调用工具的 LLM（如 Claude Sonnet），负责推理、提出假设、通过工具接口与环境交互（调用 simulate_action、predict_first_contact 等 API），生成带奖励标签的轨迹（包含思考、动作、观察）。
        - **Evolver**（进化器）：同一 LLM（但使用专门的进化提示），读取一批 Actor 的轨迹（包括奖励），通过**对比高奖励和低奖励轨迹**提取**策略技能**（标题、原理、适用条件、示例、来源种子及奖励分），并从失败轨迹中提取**错误记录**（描述、根因、纠正方法）以及**部分正确技能**（即使整体失败但中间步骤正确的推理）。
        - **Retriever**（检索器）：根据技能奖励分选择 top-M 技能和 top-N 错误记录，注入到 Actor 的系统提示中作为技能上下文。
    - **技能库更新策略**：采用**离线到在线 + 进化（Off2On + Evolution）** 最佳。离线阶段用第一批无技能轨迹初始化库；在线阶段每轮用新轨迹与旧库共同进化（合并、修订、剪枝），保持库容量上限。
    - **奖励函数**：每个轨迹的奖励 \( r(\tau) \in [-1, +1] \)，综合考虑结果（成功/失败）和效率（迭代次数）。快速成功得高分，早期放弃的重罚，但充分探索的失败反而比早期放弃惩罚轻（因为它们提供更丰富的学习材料）。
    - **跨任务技能迁移**：给定多个源任务家族的演进技能库，进化器根据目标任务的结构描述，将源技能中的抽象物理原理（如动量传递、接触几何、方向性冲量）重新锚定到目标任务中，无需任何目标任务的轨迹即可合成目标技能库，实现零样本迁移。

### 3. 实验设计：数据集/场景、基准、对比方法

- **测试平台**：Interphyre——基于 PHYRE 2D 程序化物理模拟构建的拓展基准。每个任务要求放置一个红球（位置、半径），使得仿真后达成特定目标（如绿球接触蓝球）。提供丰富的工具 API（场景检查、部分仿真、接触日志、反事实快照/恢复等），让代理可以主动实验。
- **任务难度**：8 个难度递增的关卡，其中 **Pass the Parcel** 和 **Catapult** 是最难的 Tier-2 级别，基础模型直接成功率极低（Catapult 上 Claude Sonnet 仅 2%）。
- **对比方法**：
    - **DIRECT**：仅通过场景描述一次直接调用 finish，无工具交互。
    - **ReAct**（标准基线）：每步 Thought-Action-Observation 循环，25 次迭代上限。
    - **Reflexion**：在 ReAct 基础上增加一次失败后的口头反思，再试一次。
    - **GRPO**（梯度更新基线）：使用 Group Relative Policy Optimization 对 Qwen-2.5-3B 进行微调（纯权重更新方法）。
    - **HExA**（本文方法）：多种变体（Off2On+Evolution、Online+Evolution 等）。
- **评估指标**：成功率（solve rate，%）、平均种子迭代次数（efficiency）。
- **模型**：Claude Sonnet 4.6（前沿闭源）、Qwen-2.5-{3B,7B,14B} Instruct、GPT-OSS-{20B,120B}、随机 agent。

### 6. 主要结论与发现

1. **HExA 大幅提升最难关卡的成功率**：在 Catapult 上，Claude Sonnet 4.6 从 2% 提升至 **67.3%**（最优变体达 76%），Pass the Parcel 从 0%/24% 提升至 **60%**。
2. **技能迁移有效**：仅从三个较易关卡学习到的技能，零样本迁移到 Catapult，使成功率从 8% 提升至 **44%**（+36 个点），证明技能库包含通用抽象原理而非仅记忆专用坐标。
3. **效率提升**：HExA 比 ReAct 和 Reflexion 使用更少的平均迭代次数（例如 Catapult 上 14.4 次 vs 22.9/21.2 次），约 37% 更高效。
4. **奖励标签关键**：无奖励变体相比有奖励的 HExA 性能下降（Catapult 上 50.7% vs 67.3%），说明环境反馈引导的演化优于仅靠模型先验。
5. **进化优于替换**：从旧库进化（Evolution）显著优于每次从头构建（Iterative Replacement），因为进化可保留有效技能并避免过拟合。
6. **在低样本预算下优于 GRPO**：HExA 在 50 种子预算下优于 GRPO（图8），但 GRPO 在更多种子后可通过直接奖励优化追赶，表明 HExA 适合冷启动，GRPO 适合后续精细调优。
7. **开放权重模型同样受益**：HExA 改进了所有测试的开放权重模型，如 GPT-OSS-120B 从 0% 提升至 54%。

### 7. 优点（方法或实验设计亮点）

- **完全无训练、无需外部监督**：HExA 是纯上下文自我改进框架，无需参数更新、离线数据、预言机或人工标注技能，兼容所有闭源 API 模型。
- **技能分层与组合**：技能库不仅存储粗粒度策略，还包含诊断条件和修正动作，形成层次结构，后续技能可建立在之前技能之上（如先默认放置，诊断出天花板撞击后，自动切换到 x=0.3 而非继续调半径）。
- **高效探索与迁移**：技能蒸馏后直接注入上下文，使得未来 episode 的开销降低；跨任务迁移能力证明技能包含通用物理原理。

- **引入新基准 Interphyre**：提供了丰富的工具调用 API 和反事实快照能力，为未来的主动实验研究和可解释性研究奠定了基础设施。

### 8. 不足与局限

- **环境局限**：所有实验仅在 2D 物理模拟 Interphyre 中进行，尚未推广到其他需要主动实验的领域（如科学数据分析、交互式编程、机器人操控）。论文自己承认了这一点。
- **演化器依赖模型能力**：技能库的质量受限于 LLM（Evolver）的推理和对比分析能力，对于更弱的小模型，蒸馏出的技能可能不准确或不完整。
- **领域特定成功指标**：当前使用二值成功标签加效率折扣的离散奖励，对于没有简洁成功判据的开放领域（如开放式科学发现）可能不适用。
- **计算开销**：HExA 除了 Actor 调用外，每轮还需额外调用 Evolver（用于技能蒸馏），虽然样本效率高，但在更大规模下其渐近性能天花板是否优于 GRPO 等纯参数方法尚不确定。
- **技能过拟合风险**：虽然进化策略缓解了这一问题，但在有限种子下仍可能产生过于窄化的策略，影响泛化。
- **未探索多模态输入**：基准当前仅使用文本状态描述，未集成图像或物理属性感知，可能限制了其在更实际场景中的应用。
- **仅评估有限模型**：虽覆盖了前沿和开放模型，但未在其他架构（如 Gemini、Llama 系列）上验证。
