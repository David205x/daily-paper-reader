## 摘要
自我进化框架通常优化任务方案，同时将周围的框架视为固定不变。我们引入了感知框架的自我进化（HASE），这是一个智能体强化学习框架，在该框架中，单个模型可以在多轮动作空间中生成任务方案或编辑选定的框架组件。HASE使得单个Qwen3-8B模型能够匹配使用Claude Code作为框架提出者的GPT-OSS-120B模型的文本分类性能。在alpha因子挖掘中，HASE超越了已报道的GPT-OSS-120B基线。HASE还能修复不完善的评估组件，并在圆填充算法发现中收敛到最先进的性能。这些结果表明，HASE通过一个统一的智能体过程改进了框架和方案。

## Abstract
Self-evolving frameworks usually optimize task solutions while treating the surrounding harness as fixed. We introduce Harness-Aware Self-Evolving (HASE), an agentic reinforcement-learning framework in which a single model can generate task solutions or edit selected harness components in a multi-turn action space. HASE enables a single Qwen3-8B model to match the text-classification performance of a GPT-OSS-120B model that uses Claude Code as the harness proposer. In alpha factor mining, HASE outperforms the reported GPT-OSS-120B baseline. HASE also repairs imperfect evaluation components and converges to state-of-the-art performance in circle-packing algorithm discovery. These results show that HASE improves the harness and the solution through one unified agentic process.

### 论文总结：Harness-Aware Self-Evolving (HASE)

#### 1. 核心问题与整体含义（研究动机和背景）

-   **核心问题**：当前主流的自进化系统在优化任务解决方案时，通常将外围的“框架”（Harness）视为固定不变的。这里的“框架”包括提示词、记忆、上下文检索、工具接口、评估器等一切模型与任务进行交互的组件。这种固定假设存在两个明显局限：
    1.  **不完善的评估器**：人为编写的评估器可能不完整或与真实环境不一致，导致模型在得分驱动下“作弊”而非真正解决问题。
    2.  **低效的引导**：引导组件（如提示词）可能提供较差的搜索先验，限制了模型性能。
-   **整体含义**：本文提出了一种新的范式，旨在打破“框架固定”的限制，让模型能够协同进化自身的**模型权重**、**任务解决方案**以及决定其运行环境的**框架组件**，从而在更高级别上实现自我优化。

#### 2. 方法论：核心思想、关键技术细节与算法流程

-   **核心思想**：提出 **HASE（Harness-Aware Self-Evolving）** 框架，一个统一的**智能体强化学习**框架。在该框架中，单个模型通过一个**多轮动作空间**，既可以生成任务解，也可以编辑选定的框架组件。
-   **关键技术细节**：
    1.  **两种类型的框架组件**：
        -   **引导组件（Guidance Components）**：如提示词、记忆格式化、上下文检索等，用于提高搜索质量。HASE可以自由改进它们。
        -   **评估组件（Evaluation Components）**：如验证器和评分器，用于定义方案有效性和任务指标。HASE对评估组件的编辑受到严格约束。
    2.  **统一动作空间**：模型拥有一个多轮动作空间，可以在“提交解决方案”和“编辑框架”（通过`ls`, `cat`, `grep`, `replace`等沙盒命令）之间自由选择。
    3.  **双轨制优化流程**：
        -   **内部循环（GRPO训练）**：模型在多轮轨迹中行动，最终提交解决方案并获得奖励。对引导组件的编辑通过“概念验证”（PoC）进行信用分配；对评估组件的编辑仅在代理得分与真实世界得分分歧时获得奖励。
        -   **阶段级审查（Phase-level Review）**：在每个训练阶段结束时，HASE会审查本阶段所有候选的框架编辑，选择最有效的引导组件更新进行提交。对评估组件的编辑则通过一个**错配集（Mismatch Set）** 进行锚定，该集合记录了使代理得分与真实世界得分产生分歧的案例。只有修复了错配集的编辑才会被接受，从而防止奖励破解。
-   **算法流程**：
    1.  初始化模型权重 `πθ` 和框架 `h`。
    2.  对每个阶段，进行多个 GRPO Epoch。
    3.  在每个 Epoch：根据当前框架 `h` 进行轨迹展开，模型可生成方案或编辑框架；用本地评估器评分，进行GRPO更新。
    4.  阶段结束时：进行阶段级审查，选出最好的框架更新并持久化，进入下一阶段。
