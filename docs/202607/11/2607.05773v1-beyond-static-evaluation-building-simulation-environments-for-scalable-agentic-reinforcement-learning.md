## 摘要
随着大语言模型（LLMs）演变为自主智能体，传统的静态评估无法捕捉多步决策过程。我们引入了AgenticAI-Supervisor，这是一个由API和UI驱动的RL Gym环境，它将环境创建与可扩展执行解耦。通过转向可验证的执行结果，该平台生成高保真轨迹并应用多维奖励塑形。关键的是，我们的框架通过严格的内部状态验证和测试来减轻奖励黑客攻击。这项工作通过一个客户支持智能体案例研究，展示了我们平台的核心能力，该研究展示了用于模型优化的持续闭环反馈。未来的工作将侧重于高级功能，如计算机使用、工具使用、自动化“刁难”和边缘案例生成。

## Abstract
As Large Language Models (LLMs) evolve into autonomous agents, traditional static evaluation fails to capture multi-step decision-making. We introduce AgenticAI-Supervisor, an API and UI-driven RL Gym environment that decouples environment creation from scalable execution. By moving to verifiable execution outcomes, the platform generates high-fidelity traces and applies multi-dimensional reward shaping. Critically, our framework mitigates reward hacking through rigorous internal state validation and testing. This work provides a first look at our platform's core capabilities through a Customer Support Agent case study demonstrating a consistent closed-loop feedback for model optimization. Future work will focus on advanced features such as Computer Use, Tool Use, automated "stumping", and edge-case generation.

## 2. 方法论

### 核心思想
提出**AgenticAI-Supervisor**平台，一个API和UI驱动的强化学习（RL）仿真环境，将环境创建与可扩展执行解耦，采用“运行-验证”（Run-to-Verify）闭环架构，通过严格的内部状态验证生成确定性奖励，避免奖励黑客。

### 关键技术细节

- **双阶段框架**：
  - **环境脚手架（Scaffolding）**：包含三个核心组件——领域驱动的工作流（含标准操作和故意失败状态）、基础工具模拟器（支持有状态API和Web UI，通过模型上下文协议MCP标准化接口）、数据集连接器（绑定测试用例到特定环境上下文）。
  - **可扩展执行引擎**：通过容器化无状态沙箱实现数千个并行隔离rollout；包括Rollout Handler（生命周期管理）和Agent Runtime（LLM提示、动作解析、观察检索）；所有事件记录为结构化Spans，聚合形成高保真执行轨迹（Traces）。

- **多维奖励塑形**：
  - **结果奖励（Outcome Reward）**：将终端环境状态与期望答案进行多集相等性比较，并检查资源预算约束，二进制信号，忽略自然语言输出。
  - **约束遵守（Constraint Adherence）**：三项子检查——（i）禁止字段值检验；（ii）副作用检测（比较实体计数基线）；（iii）输出忠实性（交叉引用工具API响应与智能体回复中的事实）。
  - **轨迹效率奖励（Trajectory Efficiency Reward）**：五个子组件：
    - 工具正确性：有效（非错误、非禁止）调用比例减去禁止动作惩罚。
    - 冗余调用惩罚：相同工具相同参数重复成功调用。
    - 验证错误惩罚：工具返回错误的调用比例。
    - 最小工具覆盖分数：通过平滑非线性函数结合欠覆盖和过度覆盖惩罚，并与冗余/错误惩罚进行sigmoid聚合。
    - 步数惩罚效率修正：根据执行步数与真实轨迹步数差缩放奖励，提供可微分冗长惩罚。

- **评估机制**：
  - 确定性验证器：基于状态的检查（匹配黄金答案、环境约束、API/GUI标识符交叉引用），无推理成本。
  - LLM-as-a-Judge：用于定性维度（响应连贯性、推理质量），基于结构化评分标准提供部分密级信号；采用集成判断降低变异性，最终奖励可通过配置校准。
