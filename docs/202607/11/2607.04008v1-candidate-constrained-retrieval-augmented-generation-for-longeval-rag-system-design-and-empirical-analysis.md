## 摘要
我们提出了一种面向LongEval-RAG的候选约束检索增强生成系统，其中每个查询与一个组织者提供的候选集关联，所有检索到的证据和最终引用必须局限在该集合内。该系统结合了确定性溯源追踪与基于段落的检索、确定性查询扩展、伪相关反馈（PRF）、倒数排序融合（RRF）、轻量级证据重排序、引文感知证据聚合以及可选的MiniLM句子重排序。我们使用主要组织者评估和补充的自诊断协议评估了十种流水线变体。主要评估表明，最强的平衡变体是rule-minilm：一种基于规则的块分割流水线，包含查询扩展、PRF、RRF、重排序、引文先验和后期MiniLM句子选择。该变体在我们的提交中获得了最高的BERTScore、检索精度、信息块覆盖率和平均评分。结果表明，主要增益并非来自更复杂的语义或主题转移块分割，而是来自将稳定的基于规则的证据单元与生成前的句子级神经选择相结合。补充的LLM评判评估对于早期诊断和额外分析仍然有用，但它强调的是不同于基于主要黄金答案和信息块的评估的系统，突显了多指标RAG评估的必要性。

## Abstract
We present a candidate-constrained retrieval-augmented generation system for LongEval-RAG, where each query is associated with an organizer-provided candidate set and all retrieved evidence and final citations must remain within that set. The system combines deterministic provenance tracking with passage-based retrieval, deterministic query expansion, pseudo-relevance feedback (PRF), reciprocal rank fusion (RRF), lightweight evidence reranking, citation-aware evidence aggregation, and optional MiniLM sentence reranking. We evaluate ten pipeline variants using a primary organizer evaluation and a supplementary self-generated diagnostic protocol. The primary evaluation shows that the strongest balanced variant is rule-minilm: a rule-based chunking pipeline with query expansion, PRF, RRF, reranking, citation prior, and late MiniLM sentence selection. This variant obtains the highest BERTScore, retrieval precision, nugget coverage, and average grade among our submissions. The result suggests that the main gain does not come from more complex semantic or topic-shift chunking, but from pairing stable rule-based evidence units with sentence-level neural selection before generation. The supplementary LLM-judge evaluation remains useful for early diagnosis and additional analysis, but it emphasizes different systems than the primary gold-answer and nugget-based evaluation, highlighting the need for multi-metric RAG evaluation.

## 2. 论文提出的方法论
### 核心思想
- 构建一个模块化的候选约束RAG流水线，所有操作均在候选文档集内进行。系统结合确定性来源追踪、基于段落的检索、查询扩展、伪相关反馈（PRF）、倒数排序融合（RRF）、轻量级证据重排序、引文感知证据聚合，以及可选的MiniLM句子重排序。
- 核心设计原则：**分工明确**——规则分块保持局部上下文稳定，查询扩展和RRF提供多样化的词汇候选池，后期MiniLM仅在句子层面进行神经排序，不干扰上游分块边界。

### 关键技术细节
1. **文档加载与候选约束连接**：只加载查询候选集中的文档，确保永不检索候选集外的内容。文本选择优先级：`fullText` → `abstract` → `title`。当前评估切片中，47.2%使用全文，52.8%使用仅摘要。
2. **段落构建**：三种分块策略：
   - **规则分块**：滑动句子窗口，每块最多140词，相邻块重叠1个句子。
   - **语义分块**：使用all-MiniLM-L6-v2嵌入，当候选句子与当前块均值嵌入余弦相似度≥0.72且不超词限时合并。
   - **主题转移分块**：相邻句子嵌入余弦距离>0.18时切分新块，同样受140词限制。
3. **词汇检索与查询变体**：
   - BM25基线（k1=1.5, b=0.75）。
   - **确定性查询扩展**：从原始查询中提取至多12个非停用词token（不含LLM），可构建关键词、时间意图、方法意图等变体，最多5个变体（含原始查询）。
   - **伪相关反馈**：取原始查询BM25前4篇段落，提取频次最高的6个新词（长度≥4，非停用词，不在原关键词集中），构成一个独立PRF查询变体（不混入其他变体）。
4. **排序融合**：多查询变体的BM25排名列表通过倒数排序融合（RRF）合并，参数k=60。
5. **重排序与引文先验**：取RRF前24篇段落作为短列表，重排序分数 = RRF分数 + 0.05×归一化引文图度数（文献被引/引用次数作为先验），另加时间调整项（用于有时间意图的查询）。其中引文先验是对候选文档的偏好加权。
6. **证据选择与提示构造**：选择前20篇段落（不设每篇文档上限），去重后生成`references`列表（按首次出现去重）。从每个段落提取至多5个候选句子，再选取至多10个句子级证据项。MiniLM句子重排序变体使用cross-encoder/ms-marco-MiniLM-L-6-v2对句子候选进行额外评分；非MiniLM变体保留段落选择确定的顺序。
7. **答案生成**：使用OpenAI gpt-5.4-mini，接收候选约束证据，最多5个答案句子。每个句子包含指向`references`列表的引用索引，实现确定性来源追溯。
8. **来源追踪**：每个证据项携带`evidence_id`，最终输出JSONL中每个引用句子可追溯到源文档ID。

### 公式与算法流程（文字说明）
- BM25评分：标准BM25公式，使用IDF和词频。
- RRF融合：每篇段落对每个排名列表贡献1/(60+rank)，求和得到最终RRF分数。
- 重排序分数：RRF分数 + 0.05×引文先验 + 时间调整。
- MiniLM句子重排序：在句子候选上使用交叉编码器模型进行语义相似度评分。

## 6. 主要结论与发现
1. **最强变体是rule-minilm**：在主要评估中取得了最高的BERTScore（0.168）、检索精度（0.975）、nugget覆盖率（0.367）和平均nugget等级（2.929）。ROUGE-L并非最高，但综合表现最佳。
2. **提升来源**：主要增益并非来自更复杂的语义或主题转移分块，而是来自**将稳定的基于规则的证据单元与句子级神经选择相结合**。即：上游保持确定性规则分块，后期使用MiniLM进行句子级重排序。
3. **不同指标奖励不同行为**：例如topic-shift-current获得最高ROUGE-L，topic-shift-minilm获得最高TFC1，但rule-minilm在其他关键指标上胜出。这说明不能依赖单一指标。
4. **MiniLM的作用**：在规则分块基础上，后期MiniLM句子重排序一致提升BERTScore、检索精度、nugget覆盖率等；但在语义/主题分块家族中，MiniLM的效果有混合性（某些指标提升，某些下降）。
5. **主要评估与补充评估存在分歧**：补充LLM评判评估（Claude Sonnet 4.5）将最高平均分给了rrf-no-rerank，而非rule-minilm。说明LLM评判可能更偏好流畅或看似合理的综合答案，而低估了引用精度和nugget覆盖方面的改进。因此，需要多指标评估。
6. **检索/证据选择是必要的**：所有基于检索的系统都显著优于无关检索的concat-baseline。

## 7. 优点
- **任务针对性**：针对候选约束设置的专用设计，实现了严格的引用约束和确定性来源追溯，适合分析RAG内部质量。
- **模块化流水线**：组件可单独消融，便于理解每个模块的贡献。
- **多评估协议**：既使用组织者官方指标（多维度），又设计了补充LLM评判评估，揭示了两者之间的差异，为RAG评估方法论提供了见解。
- **清晰的分工策略**：规则分块保留上下文完整性，后期MiniLM仅在句子级做神经选择，避免了错误传播。
- **代码开源**：提供了GitHub仓库，可复现。

## 8. 不足与局限
- **评估切片小**：仅47个查询，差距小的结果可能不显著，缺乏置信区间或重复实验。
- **超参数未优化**：BM25参数、分块词限、RRF常数等均为默认值，未在验证集上调优，可能影响公平比较。
- **补充评估参考答案由模型生成**：非人工标注，可能包含错误或遗漏，影响LLM评判的下游质量。
- **LLM评判的局限性**：Claude Sonnet 4.5作为评判员可能对措辞、冗长度敏感，且仍可能受到模型偏见影响。
- **文档ID重叠指标仅诊断性**：高重叠不证明答案正确使用证据，低重叠不必然错误（当多文档包含相似信息时）。
- **固定10文档候选集与全文/摘要混合输入**：限制了向其他RAG设置（如开放域、不同文档类型）的泛化能力。
- **引文网络数据（用于引文先验）可能不适用于所有查询**（例如，冷启动文档引文度数为0）。
- **时间意图处理中，时间戳未在最终批量提示中暴露**，可能限制了时间比较能力。
- **未讨论运行时效率或成本**（如API调用次数），也未讨论大规模扩展性。
