## 摘要
逆强化学习从专家演示中恢复奖励函数，但标准公式假设所有演示来自单个专家。当演示来自具有不同偏好的多个专家时，参数化方法恢复的平均奖励无法很好地拟合任何单个专家。我们实现了非参数贝叶斯逆强化学习，在奖励函数上使用狄利克雷过程先验，使得潜在奖励类型的数量可以与奖励本身联合推断。推理使用折叠吉布斯采样器，结合了用于聚类分配的中餐馆过程更新和用于奖励权重的梅特罗波利斯-黑斯廷斯更新，以及作为内部规划例程的软值迭代。我们在10x10的ObjectWorld网格上评估了两种和三种真实奖励类型。串行采样器恢复了K=2，调整兰德指数为1.000，显著优于最大熵IRL基线（ARI=0.000）。扩展到K=3表明，采样器在所有运行中正确识别了聚类数量；分配ARI为0.48-0.58反映了专家类型之间的行为重叠，这种重叠在网格实例中持续存在，揭示了在ObjectWorld上进行可靠的K=3评估需要受控的对象放置而不是随机播种。我们进一步使用Ray在HPC硬件上跨CPU核心并行化采样器，在8个工作节点上实现了4.79倍的峰值加速，并描述了状态聚合过程中使用的共识合并启发式方法带来的吞吐量与准确性的权衡。代码和容器化环境可在https://github.com/dasashreeya/np_bayes_irl获取。

## Abstract
Inverse Reinforcement Learning recovers reward functions from expert demonstrations, but standard formulations assume that all demonstrations come from a single expert. When demonstrations are pooled from multiple experts with distinct preferences, parametric methods recover an averaged reward that fits no individual expert well. We implement Nonparametric Bayesian Inverse Reinforcement Learning with a Dirichlet Process prior over reward functions, allowing the number of latent reward types to be inferred jointly with the rewards themselves. Inference uses a collapsed Gibbs sampler combining a Chinese Restaurant Process update for cluster assignments with a Metropolis-Hastings update for reward weights, and soft value iteration as the inner planning routine. We evaluate on a 10x10 ObjectWorld grid with two and three ground-truth reward types. The serial sampler recovers K=2 with Adjusted Rand Index of 1.000, substantially outperforming a Maximum Entropy IRL baseline (ARI=0.000). Extension to K=3 shows that the sampler correctly identifies the number of clusters in all runs; assignment ARI of 0.48-0.58 reflects behavioral overlap between expert types that persists across grid instantiations, revealing that reliable K=3 evaluation on ObjectWorld requires controlled object placement rather than random seeding. We further parallelize the sampler across CPU cores using Ray on HPC hardware, achieving a peak speedup of 4.79x at 8 workers, and characterize a throughput-versus-accuracy tradeoff arising from the consensus merge heuristic used during state aggregation. Code and a containerized environment are available at https://github.com/dasashreeya/np_bayes_irl.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：标准逆强化学习（IRL）假设所有专家演示来自同一单一专家。当演示由多个具有不同偏好的专家混合而成时，参数化方法（如最大熵IRL）只能恢复一个平均奖励函数，无法代表任何单个专家。
- **动机**：在实际应用中（如机器人、自动驾驶、人机交互），演示数据集往往具有异质性，需要自动发现潜在的不同奖励类型。
- **目标**：提出一种非参数贝叶斯逆强化学习方法，允许奖励函数的数量随数据自动增长，同时并行化推理以应对计算开销。

## 2. 方法论

- **核心思想**：使用**狄利克雷过程（DP）先验**对奖励函数（由权重向量参数化）建模，实现无需预先指定专家类型数量的聚类。推理采用**折叠吉布斯采样器**，交替进行聚类分配更新和权重更新。
- **关键技术细节**：
  - 马尔可夫决策过程（MDP）中奖励参数化为 `R(s) = ϕ(s)^T w`，其中ϕ是状态特征，w是权重。
  - 生成模型：`G ~ DP(α, G0)`，`wi ~ G`，`τi ~ p(τ|wi)`。G0 = N(0,I)，α为浓度参数。
  - 专家行为模型采用**玻尔兹曼理性**模型：`π(a|s) ∝ exp(β Q_w(s,a))`，Q_w通过软值迭代（100次贝尔曼备份）计算。
  - **聚类分配更新**（基于中餐馆过程）：
    ```
    p(zi = k | ·) ∝ { n_{-i,k} * p(τi|wk)   (已有k)
                    { α * p(τi|w_new)       (新簇)
    ```
  - **权重更新**：对每个簇，使用Metropolis-Hastings提出新权重 `w'_k = w_k + ε, ε~N(0,σ²I)`，接受概率为min(1, 后验比)。
  - 所有似然计算采用对数空间避免下溢。
- **并行化**：使用Ray框架，轨迹分区到多个worker，每个worker独立执行一轮吉布斯采样后合并状态。合并时基于元素级容差（10⁻⁶）识别对应聚类，导致近似。

## 3. 实验设计

- **环境**：使用**ObjectWorld** 10×10网格（|S|=100, |A|=4）。特征向量ϕ(s)∈R¹⁶（二进制，编码每个细胞最近和第二近物体的颜色和距离带）。
- **专家类型**：
  - K=2：红色爱好者（对红色正权重） vs 蓝色爱好者（对蓝色正权重），各10条轨迹，共20条。
  - K=3：增加黑色爱好者（对黑色正权重，对绿色负权重），各10条轨迹，共30条。
- **对比方法**：**最大熵IRL**（MaxEnt IRL）作为基线，单组件拟合。
- **推理超参数**：α=5.0, β=5.0, γ=0.95, σ=0.01, 吉布斯迭代500次，烧入100次。
- **评估指标**：调整兰德指数（ARI，0=随机，1=完美）、ℓ2权重误差（每簇与最近真实权重的归一化距离）、墙钟时间。

## 6. 主要结论与发现

- **非参数贝叶斯IRL能在K=2时完美恢复聚类（ARI=1.000）**，而MaxEnt基线因单组件拟合得ARI=0.000。
- **K=3时聚类计数正确，但分配准确性受限于环境特征可区分性**，而非采样器本身。蓝色和黄色物体在默认ObjectWorld中缺失，导致蓝色爱好者退化为“避免红色”信号，与黑色爱好者行为重叠。
- **并行化实现峰值加速4.79×（8 worker）**，但超线性加速（2 worker时2.76×）归因于JAX JIT编译摊销。超过8 worker后因协调开销导致速度下降。
- **吞吐量与准确性之间存在权衡**：随着worker数量增加，ARI急剧下降（16 worker时仅0.035），但ℓ2权重误差保持稳定。原因在于合并步骤中独立MH更新导致权重漂移，相同聚类被错误计数为多个簇，抬高K至16-18。
- 超参数消融表明：α=5.0、σ=0.01在K=2实验中稳健，但两者均需根据问题规模调整；σ较小时（0.001）因混合变差导致误差增大；α过大（10.0）会轻微过度碎片化。

## 7. 优点

- **方法创新性**：将狄利克雷过程先验与折叠吉布斯采样结合用于多专家IRL，实现未知专家类型数量的自动推断，解决了混合演示下的关键问题。
- **实现完整性**：提供完整开源代码（GitHub）、Docker镜像和SLURM脚本，可直接复现。
- **并行化探索**：首次系统分析Ray在IRL采样器上的加速比、准确率权衡，并指出合并步骤的改进方向。
- **实验洞察**：深入分析了ObjectWorld环境对K=3恢复的限制，揭示了特征可区分性（而非算法）是瓶颈，为基准设计提供指导意见。
- **清晰的问题隔离**：通过保持ℓ2误差稳定但ARI下降，精准定位问题在合并而非推理质量。
