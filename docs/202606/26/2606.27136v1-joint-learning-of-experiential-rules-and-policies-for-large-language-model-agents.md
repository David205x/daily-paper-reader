## 摘要
对于在多步交互环境中的大型语言模型智能体，一个关键挑战是如何有效利用累积的交互经验。现有工作通常将这种经验的两种用途分开：将其作为自然语言规则保留在模型外部以供后续提示使用，或利用轨迹和反馈更新模型参数。前者易于解释，但可能随着策略的演变而不同步；后者则更广泛地改进策略，但在稀疏奖励设置中仅对局部错误提供有限的纠正。我们提出了大型语言模型智能体的经验规则与策略联合学习（JERP），该方法从相同的交互轨迹中更新长期经验规则池和策略。在决策时，JERP检索任务相关规则，并与交互历史一起作为智能体的条件。在每个回合结束后，它利用收集到的轨迹来优化策略，并通过将当前rollout与参考成功轨迹进行比较来修改规则池。这种耦合使得规则池与不断发展的策略保持一致，同时将稳定且有效的行为逐渐吸收到模型本身中。在AlfWorld和WebShop上的实验表明，JERP在复杂交互任务中持续提升了决策性能。

## Abstract
For LLM agents in multi-step interactive environments, a key challenge is to make effective use of accumulated interaction experience. Existing work has typically separated two uses of such experience: keeping it outside the model as natural-language rules for later prompting, or using trajectories and feedback to update the model parameters. The former is easy to interpret but can fall out of sync with the evolving policy; the latter improves the policy more broadly but provides only limited correction for local mistakes in sparse-reward settings. We present Joint Learning of Experiential Rules and Policies for LLM Agents (JERP), which updates a long-term experiential-rule pool and the policy from the same interaction trajectories. At decision time, JERP retrieves task-relevant rules and conditions the agent on them together with the interaction history. After each episode, it uses the collected trajectories both to optimize the policy and to revise the rule pool by comparing current rollouts with reference successful trajectories. This coupling keeps the rule pool aligned with the evolving policy while allowing stable and effective behaviors to be gradually absorbed into the model itself. Experiments on AlfWorld and WebShop show that JERP yields consistent gains in decision performance for complex interactive tasks.

- **研究动机**：大型语言模型（LLM）在多步交互环境（如网页导航、具身交互）中面临的关键挑战是如何有效利用累积的交互经验。现有方法通常将经验用于两种分离的用途：
    - **外部化规则**：将经验转化为自然语言规则（如反思文本、指导规则、操作手册），可存储、检索和修正，作为提示上下文。优点是可解释，但可能随策略演化而不同步，甚至产生误导。
    - **参数更新**：使用轨迹和反馈直接更新模型参数（如强化学习微调）。优点是全局改进行为，但在稀疏奖励设置中对局部错误纠正不足。
- **核心问题**：如何将同一交互数据同时用于**显式规则更新**和**策略优化**，实现两者的协同进化，从而弥补各自局限。
- **整体含义**：提出JERP框架，通过联合学习规则池与策略，在决策时检索任务相关规则，在回合后同时更新规则和参数，使规则与策略对齐，并将稳定有效的行为逐渐内化到模型中。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程（文字说明）
- **核心思想**：在每个回合中，智能体使用从长期规则池中选出的工作规则集与交互历史共同作为决策上下文；回合结束后，同一批轨迹同时用于：
    - **策略参数更新**（基于组相对优势，类似GRPO）
    - **规则池更新**（通过对比当前轨迹与参考成功轨迹，生成结构化编辑操作）
- **关键技术细节**：
    - **规则池结构**：每个任务实例d维护一个长期规则池K(d)，包含若干规则（文本内容+效用分数）。
    - **工作规则集构建**：每回合开始时，按效用分数降序选择top-k条规则作为当前上下文（eK(d)）。
    - **轨迹采样**：使用当前策略πθ和选定规则集生成N条完整轨迹，得到轨迹组Td。
    - **参数更新**（第一阶段）：
        - 计算每条轨迹的累积回报R(τi)（仅终止时获得奖励）。
        - 在组内标准化回报，得到组相对优势Ai = (R(τi) - μd) / (σd + δ)。
        - 使用GRPO风格的目标函数更新策略参数，包括裁剪的替代损失和KL正则项。
    - **规则更新**（第二阶段）：
        - 输入：当前规则池K(d)、当前轨迹组Td、参考成功轨迹集T+(d)（累积的历史成功轨迹）。
        - 通过对比反思（ReflectAndEdit）生成五种结构化编辑操作：
            - ADD：添加新规则，赋初始分数s0。
            - EDIT：修改规则文本，保留原分数。
            - UPVOTE/DOWNVOTE：增加/减少规则效用分数（固定增量δ+/δ-）。
            - MERGE：合并语义相似且功能重叠的规则，新规则分数取最大值。
        - 应用操作后，若规则分数低于阈值则删除，并控制规则池容量。
- **算法流程**（Algorithm 1）：初始化规则池为空；对每个训练步骤e：采样任务d，构建工作规则集；采样N条轨迹；计算优势并更新策略参数；执行规则更新；更新参考成功轨迹集（奖励>0的轨迹加入）。

## 6. 论文的主要结论与发现
- JERP在AlfWorld上总体成功率61.5%，优于GRPO（57.8%）和RLOO（48.7%）；在WebShop上平均得分79.0、成功率64.1%，优于所有baseline。
