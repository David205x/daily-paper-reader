## 摘要
基于大语言模型（LLM）的自动规范合规（ACC）方法在处理建筑法规时，容易生成不正确且产生幻觉的计算机可处理规则。本文提出P4IR，一种两阶段框架：首先使用监督微调（SFT）向LLM灌输领域知识，然后通过组相对策略优化（GRPO）提高以高级代码骨架形式生成的中间表示的准确性。与SFT基线相比，该框架在树编辑距离和标记级Levenshtein距离上分别实现了高达23.8%和38.6%的降低。对比分析表明，在零样本设置下，该方法在代码结构和语义上均优于领先的LLM，特别是Claude Opus和Sonnet 4.5、GPT-5.2、Qwen-3-Max以及GLM-4.7（通过少样本提示评估）。此外，GRPO阶段还产生了虽小但统计显著的假阳性减少。通过将SFT与GRPO结合，直接优化领域特定目标，该方法为构建更准确、更可靠的基于LLM的ACC系统提供了路径。

## Abstract
Large language model (LLM)-based approaches for automated code compliance (ACC) of building regulations are prone to generating incorrect and hallucinated computer-processable rules. This paper introduces P4IR, a two-stage framework that uses supervised fine-tuning (SFT) to instill domain knowledge in an LLM, followed by Group Relative Policy Optimization (GRPO) to improve the accuracy of the generated intermediate representations in the form of high-level code skeletons. The framework achieved reductions of up to 23.8% and 38.6% in tree edit distance and token-level Levenshtein distance respectively, relative to the SFT baselines. Comparative analysis demonstrates that this approach in a zero-shot setting outperforms leading LLMs in both code structure and semantics, specifically Claude Opus and Sonnet 4.5, GPT-5.2, Qwen-3-Max, and GLM-4.7, evaluated via few-shot prompting. Additionally, the GRPO stage produced a small yet statistically significant reduction in false positives. By combining SFT with GRPO to optimize directly for domain-specific objectives, this approach offers a path toward more accurate and reliable LLM-based ACC systems.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：两阶段框架 **P4IR**（Policy Optimization for Intermediate Representations）：
  1. **第一阶段：SFT 知识注入** —— 使用标注的法规-代码骨架对预训练 LLM（Mistral 7B Instruct v0.3）进行监督微调，使其掌握建筑规范领域的类/函数命名与层次结构。
  2. **第二阶段：GRPO 结构优化** —— 基于 SFT 后的初始策略，通过 Group Relative Policy Optimization 进一步优化输出。GRPO 无需 critic 网络，通过从同一 prompt 采样一组候选输出，计算相对优势并更新策略，同时用 KL 散度约束防止偏离参考策略。
- **关键技术细节**：
  - **中间表示（IR）**：采用“代码骨架”——只保留类/函数定义、参数和嵌套关系，低层实现细节用 `pass` 替换，以减少幻觉风险。
  - **奖励函数设计**：
    - 主要奖励：基于 **Jaccard 相似度** 衡量生成与参考代码骨架的层次签名集对齐程度。
    - 惩罚项：语法惩罚（-1，针对无效 Python 代码），长度惩罚（-3，防止奖励 hacking）。

- **流程**：输入法规文本 → SFT 模型生成初始代码骨架 → GRPO 组采样 → 奖励计算 → 优势归一化 → 策略更新。

## 6. 论文的主要结论与发现
- **性能提升**：相比纯 SFT 基线，P4IR 使树编辑距离降低高达 **23.8%**，标记级 Levenshtein 距离降低 **38.6%**（最佳为 SFT 8 epoch + GRPO 2000 步）。

- **假阳性减少**：GRPO 显著降低了生成代码骨架中的多余类/函数，均值从 21.60 降至 18.60，统计显著（p<0.01），但效应量较小（Cohen's d=0.127）。
- **SFT 成熟度窗口**：GRPO 的有效性取决于初始 SFT 策略。过弱（<8 epoch）或过强（>10 epoch）均不利，最佳窗口在 **8～10 epoch**。Logit 熵分析表明适中熵值有利于平衡探索与利用。
- **语义保留**：GRPO 主要改善结构，CodeBERTScore 精度和召回保持稳定，说明 SFT 注入的领域语义未被破坏。

- **奖励设计**：Jaccard 相似度+语法/长度惩罚的组合有效引导模型生成结构准确、非冗余的代码骨架。

- **实用性**：零样本即可使用，无需外部检索，保护敏感数据隐私；生成的代码骨架可直接作为专家开发起点。
