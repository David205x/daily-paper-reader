## 摘要
由视觉语言模型驱动的GUI智能体在自动化数字任务方面展现出巨大潜力，但由于缺乏步骤级验证，在长周期场景中经常失败。现有的过程奖励模型仅通过文本推理来验证动作，忽略了GUI状态变化的视觉特性。我们提出VisCritic，一种视觉过程奖励框架，通过在视觉特征空间中直接比较动作前后的截图来验证智能体动作。VisCritic采用孪生视觉变换器提取变化感知表示，并结合动作感知评论头，共同评估动作成功、任务进度和错误类型。一个评论训练数据构建流程从现有轨迹中生成弱监督样本，无需额外的人工标注用于评论训练。在五个基准上的实验和离线分析表明，VisCritic可作为即插即用的增强模块用于多种GUI智能体，在提供视觉诊断线索的同时普遍提升基准指标。

## Abstract
GUI agents powered by vision-language models show strong potential for automating digital tasks, yet frequently fail in long-horizon scenarios due to the absence of step-level verification. Existing process reward models verify actions through textual reasoning alone, missing the visual nature of GUI state changes. We introduce VisCritic, a visual process reward framework that verifies agent actions by directly comparing pre-action and post-action screenshots in visual feature space. VisCritic employs a Siamese vision transformer to extract change-aware representations, coupled with an Action-Aware Critic Head that jointly evaluates action success, task progress, and error type. A critic-training data construction pipeline generates weakly supervised samples from existing trajectories without additional human labels for critic training. Experiments and offline analyses across five benchmarks demonstrate that VisCritic serves as a plug-and-play enhancement for diverse GUI agents, generally improving benchmark metrics while providing visual diagnostic cues.

- **整体含义**：论文提出 **VisCritic**，一种**视觉过程奖励框架**，通过直接比较动作前后的截图（视觉特征空间中的语义差异）来验证动作结果，弥补了文本推理的先天不足，为 GUI 智能体提供更自然的步骤级奖励信号。VisCritic 可以作为即插即用模块，在不修改智能体架构的前提下提升其长程任务成功率。

## 2. 论文提出的方法论

### 核心思想
利用 **Siamese 视觉 Transformer** 对动作前截图 \( s_t \) 和动作后截图 \( s_{t+1} \) 提取共享的 patch 级特征，计算差分特征（\( \Delta F = F_{t+1} - F_t \)），并通过**变化区域注意力**聚焦于语义上有意义的 UI 变化，最终由**动作感知评论头**（Action-Aware Critic Head）联合预测动作成功概率、任务进展分数和错误类型。

### 关键技术细节
1. **视觉差异编码器（VDE）**：
   - 共享权重的 ViT（InternViT-6B）提取 patch 特征。
   - 计算 patch 级差分和变化幅度图 \( M \in \mathbb{R}^P \)。
   - 使用带可学习温度 \( \beta \) 的 Softmax 计算变化区域注意力权重 \( \alpha \)，得到全局注意力差异向量 \( v_\Delta \)。
2. **动作感知评论头**：
   - 动作 \( a_t \) 和指令 \( l \) 由文本编码器（Qwen2.5-7B）提取特征。
   - 通过交叉注意力和空间自注意力融合视觉差异与文本上下文，经变换幅度加权池化后生成多任务输出：
     - \( \hat{y}_{suc} \)：动作成功概率（sigmoid）
     - \( \hat{y}_{prog} \)：任务进展分数（tanh 归一化到 [-1,1]）
     - \( \hat{y}_{err} \)：错误类型（5 类：成功、无操作、错误目标、页面错误、超时）
3. **评论训练数据构建**：
   - 无需额外人工标注，从现有轨迹中自动生成弱监督样本。
   - 正样本：成功轨迹中的连续三元组 \((s_t, a_t, s_{t+1})\)。
   - 负样本：四种扰动策略（动作错配、状态错配、无操作检测、失败轨迹采样）。
   - 错误类型标签由扰动策略确定性派生（与 500 例人工标注的一致性达 81.4%）。
4. **两阶段训练**：
   - 阶段1：对比预训练（InfoNCE 损失 + 可选的定位损失，5 epoch）。
   - 阶段2：多任务微调（二分类交叉熵 + MSE + 交叉熵联合损失，10 epoch）。

## 7. 优点

- **方法创新**：首次将视觉状态比较作为 GUI 智能体的过程奖励信号，解决了文本验证的本质缺陷。
- **即插即用**：无需修改智能体架构或重新训练，可作为推理时模块集成。
- **免标注数据构建**：通过弱监督自动生成训练样本，降低了规模化训练成本。
- **多任务输出**：同时提供成功概率、进展分数和错误类型，便于下游恢复策略使用。
- **可解释性**：变化区域注意力图提供直观的视觉诊断。
- **跨环境泛化**：在未见过的桌面环境（OSWorld）和 Web 环境（WebArena）上仍取得正向效果。
