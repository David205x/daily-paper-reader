## 摘要
跨多样化任务部署具备不同能力的LLM智能体需要支持异构智能体群体间知识共享的基础设施。正如搜索引擎索引人类生成的制品以辅助人类解决问题，检索系统可以组织智能体生成的制品以供智能体群体复用。我们将检索增强生成（证明了人类创作制品对单个智能体的价值）扩展为检索智能体生成的制品以支持智能体群体。具体而言，智能体轨迹编码了可复用的程序性知识，但这些制品通常在单次使用后被丢弃或仅由生成智能体保留，迫使新实例化的智能体反复重新发现已有解决方案。我们提出多智能体交互记忆（MATM），一种面向群体级存储与检索智能体生成轨迹的框架，其中生产者智能体将轨迹贡献到共享仓库，消费者智能体检索这些轨迹以改进任务执行。我们聚焦交互式环境（ALFWorld与WebArena），其中轨迹较长且编码了特别丰富的程序性结构。实验表明，从MATM检索轨迹能提升下游任务性能并减少交互步骤，无需协调或联合训练。这些结果将MATM定位为开放智能体生态系统中群体级经验共享的设计模式。

## Abstract
The decentralized deployment of LLM agents with diverse capabilities across diverse tasks motivates infrastructure for knowledge sharing across heterogeneous agent populations. Just as search engines index human-generated artifacts to support human problem solving, retrieval systems can organize agent-generated artifacts for reuse across agent populations. We extend retrieval-augmented generation - which demonstrates the value of human-authored artifacts to individual agents - to retrieval of agent-generated artifacts supporting a population of agents. In particular, agent trajectories encode reusable procedural knowledge, yet these artifacts are typically discarded after a single use or retained only by the producing agent, forcing newly instantiated agents to repeatedly rediscover existing solutions. We propose Multi-Agent Transactive Memory (MATM), a framework for population-level storage and retrieval of agent-generated trajectories, where producer agents contribute trajectories to a shared repository and consumer agents retrieve them to improve task execution. We focus on interactive environments (ALFWorld and WebArena), where trajectories are long and encode especially rich procedural structure. Our experiments demonstrate that retrieving trajectories from MATM improves downstream task performance and reduces interaction steps without coordination or joint training. These results position MATM as a design pattern for population-level experience sharing in open agent ecosystems.

## 论文提出的方法论：核心思想、关键技术细节与算法流程

-   **核心思想：多智能体交互记忆（Multi-Agent Transactive Memory， MATM）**
    - 借鉴了心理学中“交互记忆”（Transactive Memory）的概念：人类群体通过将知识分布在个体之间，并利用共享机制（知道谁能做什么）来定位和检索相关信息，从而协作完成任务。
    - MATM将这一概念应用到AI智能体系统中。它构建了一个**共享的、群体级别的轨迹知识库**。智能体在生态系统中扮演两种角色：
        - **生产者（Producer Agent）**：成功完成任务后，将自己的交互轨迹贡献到共享仓库。
        - **消费者（Consumer Agent）**：开始新任务时，从共享仓库检索与当前任务和状态相关的高质量轨迹，作为参考和指导。
        - 角色的不互斥性：一个智能体可以在不同情境下既是生产者又是消费者。

-   **关键技术细节：轨迹索引与检索**
    1.  **状态条件键值索引**：
        - 对长轨迹进行分块（chunking），采用一种“状态条件”的索引方式。键(key)由任务描述和最近`l`步的交互历史（`τ`）组成，值(value)是接下来的`l`步动作序列。这使得消费者能基于自己的当前状态进行检索，而非仅依赖原始任务描述。
    2.  **检索与重排序（Learning to Rank Trajectories, LTRT）**：
        - **第一级检索**：使用一个嵌入函数（本文使用E5-Base模型）将查询和轨迹块编码为向量，进行稠密检索，返回一个候选列表。
        - **第二级重排序**：为了提高检索质量，引入了一个轻量级的学习排序（LTR）模型。该模型通过一个特征映射`ϕ(q, d)`来捕获多种特征，包括：
            - **生产者元数据**：如生产者的综合能力基准分数（如AAI指数），用于建模“信任度”。
            - **消费者元数据**：如消费者ID，用于实现个性化。
            - **轨迹特征**：如轨迹长度、是否成功等。
            - **查询-轨迹交互特征**：如文本/嵌入相似度、重叠率等。
    3.  **训练数据生成**：
        - 训练LTR模型需要“轨迹的帮助程度（边际效用）”作为标签。通过在任务执行过程中，比较有/无检索轨迹辅助下的任务完成分数差值`（s(j) - s_base）`来生成训练数据。这个过程与记忆库的增量构建同步进行。

-   **算法流程**：
    1.  **预填充（Pre-population）**：使用公开的、由较强模型（如GPT-4-Turbo）生成的成功轨迹，初始化MATM索引。
    2.  **增量构建（Incremental Update）**：
        1.  处理一系列训练问题。
        2.  为每个问题分配一个生产者智能体。
        3.  该智能体首先**不**使用MATM执行任务，得到基线轨迹和分数（`s_base`）。
        4.  在基线的若干步（分支点）后，智能体“回滚”（roll-in）到该状态，然后先从MATM检索最佳的K个轨迹块。
        5.  智能体在`I`个不同排名的轨迹块（例如1、5、10、15、20名）的指导下，分别“展开”（roll-out）完成任务，得到新轨迹和分数（`s(j)`）。
        6.  **如果新轨迹成功**，则贡献回MATM，用于更新索引。
        7.  **记录训练数据**：将`(查询, 轨迹块, 边际效用)`作为LTR模型的一个训练样本。
    4.  **推理时**：消费者智能体配备一个“检索规划器”（Retrieval Planner），用于决定当前步骤是否需要检索（避免无效检索稀释上下文）。当需要检索时，经过LTR重排序后，选择一个最佳轨迹块作为上下文指导。

## 实验设计

## 论文的主要结论与发现

1.  **有效性与效率提升是成立的**：MATM的使用（尤其是结合LTR重排序）能显著提升下游任务的成功率，并减少完成任务所需的交互步数（即提升效率）。
    - 在ALFWorld上，最佳重排序模型（SVMRank）将成功率从47.1%提升至64.3%，平均步数从11.8下降至10.3。
    - 在WebArena上，提升幅度较小，但趋势一致（SR从18.2%提升至20.5%）。
2.  **学习排序（LTR）有效**：经过训练的LTR重排序器（特别是SVMRank）优于简单的稠密检索，说明特征工程（尤其是生产者元数据）对提高检索质量至关重要。
3.  **收益分布广泛**：MATM的收益并非仅仅来源于强智能体对弱智能体的“知识传授”。即使是能力相当的智能体对，生产者也能为消费者带来价值。这表明MATM的价值超越了简单的“能力转移”。
4.  **跨任务泛化能力强**：即使限制只检索与当前任务类型不同的轨迹，对任务性能仍有提升（ALFWorld中SR从47.1%提升至59.9%）。说明轨迹中包含了可跨任务复用的通用交互模式。
5.  **可扩展性**：随着MATM仓库的增大，消费者智能体的性能（成功率）呈现持续或最终的上升趋势，证明系统的价值随规模增长而提升。

## 优点

1.  **方法设计亮点**：
    - **轻量级**：核心的LTR重排序器是一种小型、高效的模型，无需对底层LLM进行昂贵的微调或联合训练，降低了实现门槛。
    - **生态友好**：基于“生产者-消费者”的协作模式，天然支持开放的、动态变化的异构智能体生态系统，无需全局协调。
    - **个性化与信任建模**：LTR特征中包含生产者和消费者元数据，为未来的个性化检索和生产者信任度建模提供了框架基础。
2.  **实验设计亮点**：
    - **实验充分全面**：不仅比较了最终性能，还深入分析了收益的分布、跨任务泛化能力和可扩展性，结论更有说服力。
    - **消融实验严谨**：通过容量差距分析和检索范围消融，有力地反驳了“仅靠强智能体转移”的简单解释，揭示了跨任务泛化等更深层的价值。

## 不足与局限

1.  **实验覆盖范围有限**：
    - 仅使用了两个交互式环境（ALFWorld和WebArena）和34个消费者模型。无法断定在更广泛的、如代码生成、数学推理等非交互式领域或更多模型上的表现。
    - 训练和评估都在同一基准内部进行，跨基准泛化的重排序（LTR）能力未被测试。
2.  **训练标签获取成本较高**：
    - LTR模型的训练依赖于在多个分支点、对多个候选轨迹进行全量roll-out来获取边际效用标签，这个过程本身就需要大量计算，存在潜在的标签偏差风险。
3.  **检索质量依赖嵌入模型**：
    - 第一级检索使用的E5-Base嵌入模型是固定的。这种方法的效果高度依赖于该嵌入模型能否很好地捕捉任务相关的语义信息，尤其对复杂的交互轨迹编码可能不够精准。
4.  **对抗性生产者风险未考虑**：
    - 论文明确指出，当前框架未考虑恶意或错误的生产者可能贡献有害或低质量的轨迹，对消费者造成潜在风险。这在实际应用中是一个重要的缺陷。
5.  **数据集可能存在的偏置**：
    - 论文在附录D和E中描述了ALFWorld和WebArena的数据集划分，虽然作者确保了分布的相似性，但WebArena的测试集（88个）和训练集（724个）规模差异较大，且某些测试任务可能缺乏同类型轨迹，限制了部分分析（如“同任务”检索泛化实验）的结论范围。
